Topic 12: JavaScript & the Document Object Model (DOM)

Topic Progress: 80% Complete

1. What is JavaScript?

Analogy — The Three Languages of the Web: Think of building a website like constructing a house. HTML is the structure — the walls, floors, and roof. CSS is the appearance — the paint colours, furniture styles, and decorations. JavaScript is the electricity and plumbing — it makes things happen. Lights turn on when you flip switches, water flows when you turn taps, the garage door opens when you press a button. JavaScript brings your static house to life.

JavaScript is a cross-platform, object-oriented scripting language that makes web pages interactive. It powers:

  • Complex animations and visual effects
  • Clickable buttons and interactive forms
  • Popup menus and modal dialogs
  • Real-time content updates without page reloads
  • Games and multimedia applications

Client-Side vs Server-Side

Aspect Client-Side JavaScript Server-Side JavaScript (e.g., Node.js)
Runs on The user's browser The web server
Purpose Controls the browser and DOM, responds to user actions Communicates with databases, reads/writes files, handles HTTP requests
Examples Form validation, animations, dynamic content API servers, real-time chat backends, file processing

2. JavaScript Features & Characteristics

Click to expand: Key Features of JavaScript
  • Interpreted, not compiled — The browser reads and executes JS line by line (no compilation step)
  • Not strongly typed — Variables can hold any type of data without declaring the type
  • Dynamic subprograms — Functions can be added at runtime
  • Direct execution — Code runs immediately in the browser
  • Associative arrays — Objects that work like key-value dictionaries
  • Regular expressions — Built-in pattern matching for text
  • String processing — Rich built-in methods for manipulating text
  • Objects and classes — Object-oriented programming support
  • Inheritance — Objects can inherit properties from other objects
  • Case-sensitive — myVar and myvar are different variables
JavaScript is NOT Java! Despite the similar name, JavaScript and Java are completely different languages. They share some C-like syntax (if/for/while structures), but that's where the similarity ends. The name "JavaScript" was a marketing decision when it was created in 10 days — it had nothing to do with Java.

3. JavaScript Syntax

Including JavaScript in HTML

There are three ways to include JavaScript in an HTML page:

1. Inline (in the body)

<body>
  <script>
    alert('Hello, World!');
  </script>
</body>

2. In the head

<head>
  <script>
    // JavaScript goes here
  </script>
</head>

3. External file (recommended)

<!-- At the end of the body, just before </body> -->
  <script src="myscript.js"></script>
</body>
Best Practice: Place <script> tags at the end of the body, just before </html>. This lets the HTML and CSS load and render first, making the page feel faster. The user sees the content before JavaScript starts processing.
Analogy — Script Placement: Think of it like a restaurant. HTML and CSS are the food and presentation — they should be served to the customer first. JavaScript is the entertainment (live music, table-side cooking) — it starts after the customer is seated and has their menu. If you start the entertainment before the food is ready, the customer has a bad experience.

4. Selecting HTML Elements

JavaScript can find and select HTML elements using querySelector() and querySelectorAll(). These use CSS selectors — the same syntax you already know from CSS!

// Select the FIRST matching element
document.querySelector('h1')         // First <h1> element
document.querySelector('.header')    // First element with class "header"
document.querySelector('#demo')      // Element with id "demo"
document.querySelector('p.intro')    // First <p> with class "intro"

// Select ALL matching elements (returns a NodeList)
document.querySelectorAll('p')       // All <p> elements
document.querySelectorAll('.item')   // All elements with class "item"
Analogy — querySelector is like GPS: querySelector() is like a GPS that finds one specific address — it stops at the first match. querySelectorAll() is like a GPS that finds all branches of a chain — it returns every matching location. CSS selectors are the "address format" you use to tell the GPS where to look.

5. Displaying Output

JavaScript has several ways to display information to the user or developer:

Method Where It Displays Use Case
document.write() Directly into the HTML page Quick testing (not recommended for production)
element.innerHTML Inside a specific HTML element Updating page content dynamically
console.log() Browser's developer console Debugging — invisible to users
window.alert() Popup alert dialog Important notifications to the user 
// Write directly to the page (overwrites everything!)
document.write('Hello!');

// Update a specific element's content
document.querySelector('#output').innerHTML = 'New content here';

// Log to developer console (for debugging)
console.log('This value is:', someVariable);

// Show a popup alert
window.alert('Warning! Something happened.');
Avoid document.write() in real projects. If called after the page has loaded, it will overwrite the entire page. Use innerHTML or textContent to update specific elements instead.

6. Variables

Analogy — Variables are Labeled Boxes: Imagine you have a storage room with labeled boxes. Each box has a name label (variable name) and can hold different items (values). You can put a number in one box, a word in another, or even a list of items. The label lets you find and use what's inside later. In JavaScript, variables are these labeled boxes — they store data you need to reference later.

Variables in JavaScript are declared with the var keyword:

var name = 'Alice';      // String
var age = 25;            // Number
var isStudent = true;    // Boolean
var grades = [90, 85, 92]; // Array

Variable Naming Rules

Rule Valid ✅ Invalid ❌
Must begin with a letter, underscore (_), or dollar sign ($) _name, $total, userName 2name, -age
Cannot contain spaces firstName first Name
Cannot be a reserved word myVar var, function, return
Case-sensitive myVar and myvar are different Using them interchangeably
Dynamic Typing: Unlike languages like Java or C++, you don't declare what type a variable holds. JavaScript figures it out at runtime. The same variable can even change types: var x = 5; then x = 'hello'; is perfectly valid (though not recommended).

// Comparisons return booleans
var result = 10 > 5;   // true
var check = 3 === 7;   // false

Arrays

Analogy — Arrays are Numbered Lists: An array is like a numbered to-do list. Item #0 is "Buy groceries", item #1 is "Walk the dog", item #2 is "Study JavaScript". You access items by their position number, and you can add, remove, or change items. 
var fruits = ['Apple', 'Banana', 'Cherry'];

// Access items (index starts at 0!)
console.log(fruits[0]);  // 'Apple'
console.log(fruits[2]);  // 'Cherry'

// Loop through an array
fruits.forEach(function(fruit) {
  console.log(fruit);
});

// Add an item
fruits.push('Date');

console.log(fruits.length);  // 4

The typeof Operator

typeof 'hello'      // 'string'
typeof 42           // 'number'
typeof true         // 'boolean'
typeof [1, 2, 3]    // 'object' (arrays are objects!)
typeof undefined    // 'undefined'

8. Popup Boxes

Analogy — The Three Receptionists: alert() is the receptionist who just tells you something and waits for you to acknowledge it. confirm() is the one who asks a yes/no question. prompt() is the one who asks for your information and writes it down. All three stop everything else until you respond — they're "modal" (blocking) interactions.
Popup Type Syntax Returns Use Case
alert() alert('Message'); Nothing (void) Simple notifications
confirm() confirm('Are you sure?'); true (OK) or false (Cancel) Yes/No decisions
prompt() prompt('Enter your name:'); The typed string, or null if Cancelled Getting user input 
// Simple alert
alert('Welcome to my website!');

// Confirm returns true or false
var agreed = confirm('Do you agree to the terms?');
if (agreed) {
  console.log('User agreed.');
} else {
  console.log('User declined.');
}

// Prompt returns the user's input
var userName = prompt('What is your name?');
if (userName !== null) {
  alert('Hello, ' + userName + '!');
}

9. Functions

Analogy — Functions are Recipes: A recipe is a reusable set of instructions. You write it once, then follow it whenever you want to make that dish. You can customize it by changing ingredients (parameters). A function works the same way: you define the instructions once, then "call" (execute) the function whenever you need it, passing in different data each time.

Function Syntax

// Defining a function
function greet(name) {
  return 'Hello, ' + name + '!';
}

// Calling the function
var message = greet('Alice');
console.log(message);  // 'Hello, Alice!'

// Function without parameters
function sayGoodbye() {
  console.log('Goodbye!');
}
sayGoodbye();

Parts of a Function

Part Example Purpose
function keyword function Tells JavaScript you're defining a function
Name greet What you call it by
Parameters (name) Placeholder variables for input data
Body { ... } The instructions to execute
return return value; Sends a result back to whoever called the function

10. Objects

Analogy — Objects are Profile Cards: Imagine a profile card (like a contact card or employee badge). It has properties like name, phone number, and department. It can also have methods — actions like "call()" or "email()". A JavaScript object works exactly the same way: properties hold data, methods perform actions. And the this keyword is like saying "this card" — referring to the current object's own properties. 
// Creating a singleton object
var person = {
  name: 'Alice',          // Property (key-value pair)
  age: 25,                // Property
  occupation: 'Developer',// Property

  // Method (function as a property)
  introduce: function() {
    return 'Hi, I am ' + this.name + ', age ' + this.age;
  }
};

// Accessing properties
console.log(person.name);           // 'Alice'
console.log(person['occupation']);   // 'Developer'

// Calling a method
console.log(person.introduce());    // 'Hi, I am Alice, age 25'
The this keyword: Inside an object's method, this refers to the object itself. So this.name means "this object's name property". It's how an object refers to its own data from within its methods.

11. Events

Analogy — Events are Doorbells: Events are like doorbells in your house. You don't constantly check the door — you install a doorbell and set up a handler (you go answer it when it rings). Similarly, you don't constantly check if the user clicked something. You attach an event handler, and JavaScript runs that code when the event happens. The doorbell rings → you answer. The button is clicked → your code runs.

Events are things that happen to HTML elements. You can respond to them with JavaScript:

Event When It Fires Common Use
onclick Element is clicked Buttons, links, any clickable element
onchange Element's value changes Input fields, select dropdowns
onmouseover Mouse hovers over element Hover effects, tooltips
onmouseout Mouse leaves element Reverting hover effects
onkeydown User presses a key Keyboard shortcuts, search-as-you-type
onload Page or resource finishes loading Initialization code, animations 
<!-- Inline event handler -->
<button onclick="alert('Clicked!')">Click Me</button>

<!-- Better: Event handler in script -->
<button id="myBtn">Click Me</button>

<script>
  document.querySelector('#myBtn').addEventListener('click', function() {
    alert('Button was clicked!');
  });
</script>
addEventListener() is preferred over inline onclick attributes because it separates HTML from JavaScript, allows multiple handlers per event, and gives you more control (like removing handlers later).

12. The Document Object Model (DOM)

Analogy — The DOM is a Family Tree: When a web page loads, the browser creates an invisible family tree of the page. document is the great-grandparent. <html> is the grandparent. <head> and <body> are the parents. All other elements are children, grandchildren, etc. JavaScript can walk along this tree, find any "family member" (element), and change them — update their content, move them, remove them, or add new members.

The DOM is a programming interface for web documents. It is not a language — it's a model (a structured representation) that JavaScript uses to interact with the page.

DOM Tree Structure

The DOM Tree

document
  └── html
      ├── head
      │   ├── meta
      │   ├── title
      │   └── link (CSS)
      └── body
          ├── nav
          ├── h1
          ├── p
          ├── div
          │   ├── h2
          │   └── p
          └── script

DOM and JavaScript Together

The DOM provides the model; JavaScript provides the manipulation. Together they can:

  • Change HTML element content and attributes
  • Change CSS styles dynamically
  • Add new HTML elements to the page
  • Remove existing HTML elements
  • React to user events (clicks, key presses, etc.)

13. Common DOM Operations

Selecting Elements

// Select single element
var heading = document.querySelector('h1');
var box = document.querySelector('#myDiv');
var first = document.querySelector('.highlight');

// Select multiple elements
var allItems = document.querySelectorAll('.item');

Changing Content

// Change the text inside an element
heading.innerHTML = 'New Heading Text';

// Change styles directly
heading.style.color = 'blue';
heading.style.fontSize = '24px';

Creating and Adding Elements

// Create a new element
var newDiv = document.createElement('div');

// Create text for it
var textNode = document.createTextNode('Hello, I am new here!');

// Add the text to the element
newDiv.appendChild(textNode);

// Add the element to the page
document.body.appendChild(newDiv);

Removing Elements

// Select the element to remove
var oldItem = document.querySelector('#oldItem');

// Remove it from its parent
oldItem.parentNode.removeChild(oldItem);

Working with Attributes

// Set an attribute
var img = document.querySelector('img');
img.setAttribute('src', 'new-photo.jpg');
img.setAttribute('alt', 'A beautiful photo');

// Get an attribute
var src = img.getAttribute('src');
console.log(src);  // 'new-photo.jpg'

node A general term for anything in the DOM tree (elements, text, comments) textNode.parentNode
nodeList A collection of nodes (like an array) returned by querySelectorAll document.querySelectorAll('p')

Analogy — DOM Data Types as a Library: document is the entire library building. element is a specific book on a shelf. node is anything in the library (book, shelf, label, sign). nodeList is a list of search results showing multiple items. attr is the metadata card for a book (title, author, ISBN). window is the entire city block that contains the library.

15. Putting It All Together — Complete Example

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>JavaScript & DOM Demo</title>
</head>
<body>

  <h1 id="title">My To-Do List</h1>

  <input type="text" id="taskInput" placeholder="Enter a task...">
  <button id="addBtn">Add Task</button>

  <ul id="taskList"></ul>

  <script>
    // Select elements
    var input = document.querySelector('#taskInput');
    var addBtn = document.querySelector('#addBtn');
    var taskList = document.querySelector('#taskList');

    // Add task when button is clicked
    addBtn.addEventListener('click', function() {
      var taskText = input.value.trim();

      if (taskText === '') {
        alert('Please enter a task!');
        return;
      }

      // Create new list item
      var li = document.createElement('li');
      li.textContent = taskText;

      // Click to remove
      li.addEventListener('click', function() {
        taskList.removeChild(this);
      });

      // Add to the list
      taskList.appendChild(li);

      // Clear the input
      input.value = '';
    });
  </script>

</body>
</html>
This example demonstrates: selecting elements (querySelector), event handling (addEventListener), creating elements (createElement), setting content (textContent), adding elements (appendChild), removing elements (removeChild), and getting user input (input.value).

16. Summary

Key Takeaways

  • JavaScript is an interpreted, case-sensitive, object-oriented language that brings web pages to life.
  • It is NOT related to Java despite the similar name — they just share C-like syntax.
  • Variables are declared with var and can hold any data type (dynamically typed).
  • Data types include Strings, Numbers, Booleans, and Arrays.
  • Functions are reusable blocks of code that accept parameters and return values.
  • Objects have properties (key-value pairs) and methods (functions), using this to reference themselves.
  • Events let JavaScript respond to user actions like clicks, key presses, and hover.
  • The DOM is a tree-shaped model of the page that JavaScript uses to find and manipulate elements.
  • Use querySelector to find elements, createElement/appendChild to add them, and removeChild to remove them.

17. Flashcards

What is JavaScript and what does it do on a web page?
JavaScript is an interpreted, object-oriented scripting language that makes web pages interactive. On the client side, it controls the browser and DOM — responding to user events, updating content, and creating animations.
What is the difference between querySelector and querySelectorAll?
querySelector() returns the first matching element. querySelectorAll() returns all matching elements as a NodeList (collection).
How do you declare a variable in JavaScript?
Using the var keyword: var myVariable = value; JavaScript variables are dynamically typed — you don't specify the data type.
What does the this keyword refer to inside an object method?
this refers to the current object — the object that owns the method being executed. For example, in person.introduce(), this.name refers to person.name.
What is the DOM?
The Document Object Model is a tree-shaped programming interface that the browser creates when a page loads. It represents every HTML element as an object that JavaScript can find, create, modify, or remove.
What does prompt() return if the user clicks Cancel?
null. If the user types something and clicks OK, it returns the typed string. If they click Cancel, it returns null.
What are the three main data types in JavaScript?
String (text in quotes), Number (integers or decimals), and Boolean (true/false). Arrays and Objects are also important compound types.
How do you create and add a new element to the page using the DOM?
1. document.createElement('tag') to create it
2. document.createTextNode('text') for content (optional)
3. parent.appendChild(child) to add it to the page
Why should you place <script> tags at the end of the body?
So HTML and CSS load and render first, making the page feel faster. The user sees content before JavaScript starts processing.
What is the difference between alert(), confirm(), and prompt()?
alert() shows a message (returns nothing). confirm() asks yes/no (returns true/false). prompt() asks for text input (returns the string or null if cancelled).
