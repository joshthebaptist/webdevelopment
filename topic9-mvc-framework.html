1. CRUD Operations

Analogy — A Filing Cabinet: Think of a database as a filing cabinet. Creating a record is like adding a new folder. Reading is looking at what is inside a folder. Updating is rewriting a page inside a folder. Deleting is throwing the folder away. Every application that stores data performs these four actions — nothing more, nothing less.

What is CRUD?

CRUD stands for the four basic operations performed on persistent storage (like a database):

CRUDDescriptionSQL EquivalentHTTP Method
CreateAdd a new recordINSERT INTOPOST
ReadRetrieve / view recordsSELECTGET
UpdateModify an existing recordUPDATEPOST / PUT
DeleteRemove a recordDELETE FROMPOST / DELETE

CRUD Variants

Some extended forms of CRUD are commonly used:

  • SCRUD — Search + CRUD. Adds search/filter functionality to the Read operation so users can find specific records.
  • CRUDL — CRUD + Listing. Adds paginated listing of records, which is essential when you have hundreds or thousands of rows.
Tip: Almost every web application you use — from social media to e-commerce — is built on CRUD. When you sign up (Create), view your feed (Read), edit your profile (Update), or remove a post (Delete), you are using CRUD operations.

2. Building CRUD in PHP

Three Approaches to Building CRUD

Approach 1: Separate PHP Files per Operation

Each CRUD operation has its own pair of files — a UI page (HTML form) and a processing page (PHP script that handles the logic).

create.php        → HTML form for creating a record
create_process.php → PHP script to insert into DB
read.php           → Display a single record
update.php         → HTML form for editing
update_process.php → PHP script to update DB
delete_process.php → PHP script to delete from DB

This is the simplest approach but leads to many files and duplicated connection code.

Approach 2: All Operations in One Function File

All CRUD logic lives in a single file of functions. Each form page includes this file and calls the relevant function.

functions.php    → contains create_record(), read_record(), etc.
form_page.php    → includes functions.php, calls create_record()

Better code reuse, but still procedural and hard to scale.

Approach 3: OOP PHP with MVC Structure

Use Object-Oriented PHP organised into Models, Views, and Controllers. This is the professional standard and what frameworks help you build.

Models/    → User.php (data & business logic)
Views/     → user_form.php (HTML presentation)
Controllers/ → UserController.php (handles input, coordinates)
Analogy — Restaurant Kitchen: Approach 1 is like having a separate chef for every dish who also serves the food — chaotic. Approach 2 is one chef who does everything — better but exhausting. Approach 3 is a proper kitchen: chefs (Controllers) take orders, prep cooks (Models) prepare ingredients, and servers (Views) deliver plates. Each role is clear and professional.

3. Object-Oriented PHP (OOPHP)

Why OOP Over Procedural?

Procedural PHP writes code as a sequence of instructions. OOPPHP organises code into objects that bundle data and behaviour together. Here are the key advantages:

AdvantageExplanation
Faster ExecutionOOP code can be optimised and cached more efficiently by the engine
Clear StructureCode is organised into logical classes that mirror real-world concepts
DRY CodeDon't Repeat Yourself — write once, reuse everywhere via inheritance
Easier MaintenanceChanges in one class propagate automatically to all users of that class
Easier DebuggingProblems are isolated to specific classes rather than scattered across files
ReusabilityBuild a class once and use it across multiple projects or modules
Analogy — Blueprints vs. Random Lumber: Procedural code is like throwing random planks of wood together to build a shed. OOP is like having a blueprint (class) from which you can produce identical houses (objects). Each house has its own paint colour and furniture (properties) but shares the same floor plan (methods).

4. OOP Terminology

Class
A blueprint or template that defines the properties (data) and methods (behaviour) that objects of that type will have. Example: a Car class defines that every car has a colour and can drive.
Object
A specific instance of a class. If Car is the class, then $myCar = new Car() creates an object — a real car with actual values.
Member Variable (Property)
A variable defined inside a class. It stores data specific to each object. Example: public $colour = "red";
Member Function (Method)
A function defined inside a class. It defines the behaviour of objects. Example: public function drive() { ... }
Constructor
A special method (__construct()) that runs automatically when you create a new object. Used to initialise properties.
Destructor
A special method (__destruct()) that runs when the object is destroyed. Used for cleanup tasks like closing database connections.
Parent / Child Class
A parent class (also called base/super class) is inherited from. A child class (derived/sub class) extends the parent, gaining its properties and methods.
Inheritance
The mechanism by which a child class acquires the public and protected properties/methods of its parent class using the extends keyword.
Polymorphism
The ability for different classes to be treated through the same interface. A Shape method can accept a Circle, Square, or Triangle — each responds differently to draw().
Encapsulation
Bundling data and methods together and restricting direct access to some components. Achieved via private properties with public getter/setter methods.
Data Abstraction
Hiding complex implementation details and showing only the essential features. The user of a Car object calls start() without knowing how the engine works internally.
Overloading
Defining multiple methods with the same name but different parameters. PHP handles this differently from languages like Java — using magic methods.

5. Class Basics in PHP

Anatomy of a Class

<?php
class Car {
    // Member variables (properties)
    public $colour;
    public $speed = 0;

    // Constructor - runs when object is created
    public function __construct($colour) {
        $this->colour = $colour;   // $this refers to the current object
    }

    // Member function (method)
    public function accelerate($amount) {
        $this->speed += $amount;
        echo "The $this->colour car is now going $this->speed km/h<br>";
    }

    public function brake() {
        $this->speed = 0;
        echo "The car has stopped.<br>";
    }
}

// Creating objects (instances)
$myCar = new Car("red");
$yourCar = new Car("blue");

$myCar->accelerate(60);   // "The red car is now going 60 km/h"
$yourCar->accelerate(80); // "The blue car is now going 80 km/h"
$myCar->brake();          // "The car has stopped."
?>
Analogy — Cookie Cutter: The class is the cookie cutter shape. The objects are the individual cookies you press out. Each cookie (object) has its own icing colour and sprinkles (properties), but they all share the same shape (methods).

Properties: var, public, private, protected

Properties are variables inside a class. In modern PHP, always use access modifiers instead of var:

<?php
class Employee {
    public $name;         // Accessible from anywhere
    private $salary;      // Accessible ONLY inside this class
    protected $department; // Accessible inside this class AND child classes

    public function __construct($name, $salary, $department) {
        $this->name = $name;
        $this->salary = $salary;
        $this->department = $department;
    }

    // Public method to access private data (getter)
    public function getSalary() {
        return $this->salary;
    }

    // Public method to modify private data (setter)
    public function setSalary($newSalary) {
        if ($newSalary > 0) {
            $this->salary = $newSalary;
        }
    }
}

$emp = new Employee("Alice", 5000, "Engineering");
echo $emp->name;        // OK - public
// echo $emp->salary;   // ERROR - private
echo $emp->getSalary();  // OK - uses public getter
?>

Access Modifiers Summary

ModifierSame ClassChild ClassOutside Class
publicYesYesYes
protectedYesYesNo
privateYesNoNo
Analogy — House Access: Public = front door, anyone can walk in. Protected = family rooms, only family (children) and you can enter. Private = your bedroom, only you can enter. This is exactly how access modifiers work for classes and their children.

6. Inheritance

The extends Keyword

Inheritance lets a child class acquire properties and methods from a parent class. The child inherits all public and protected members. Private members are NOT inherited (they exist but cannot be accessed directly).

<?php
// Parent class
class Animal {
    public $name;
    protected $sound;

    public function __construct($name, $sound) {
        $this->name = $name;
        $this->sound = $sound;
    }

    public function speak() {
        return "$this->name says $this->sound!";
    }
}

// Child class - inherits from Animal
class Dog extends Animal {
    public function __construct($name) {
        parent::__construct($name, "Woof"); // Call parent constructor
    }

    // Child can add its own methods
    public function fetch($item) {
        return "$this->name fetches the $item!";
    }
}

class Cat extends Animal {
    public function __construct($name) {
        parent::__construct($name, "Meow");
    }
}

$dog = new Dog("Rex");
$cat = new Cat("Whiskers");

echo $dog->speak();          // "Rex says Woof!"
echo $dog->fetch("ball");    // "Rex fetches the ball!"
echo $cat->speak();          // "Whiskers says Meow!"
?>
Analogy — Family Traits: Inheritance in OOP is like real family inheritance. A child inherits eye colour and height range (public/protected) from their parents. They also get a private diary (private) that nobody else can read directly. The child can also develop new skills (new methods) that the parent doesn't have.

7. The MVC Pattern

What is MVC?

MVC stands for Model - View - Controller. It is a design pattern that separates an application into three interconnected components. This separation makes code cleaner, easier to test, and simpler to maintain.

The Three Components

Model (The Data Layer)
  • Manages all data, logic, and rules of the application
  • Handles database operations (CRUD)
  • Contains business rules and validation
  • Notifies the View when data changes
  • Knows nothing about the UI — it just manages data
View (The Presentation Layer)
  • Displays data to the user (HTML, CSS)
  • Receives data from the Controller
  • Contains the user interface
  • May contain minimal logic for display formatting
  • Knows nothing about database queries — it just renders data
Controller (The Coordinator)
  • Handles user input (form submissions, URL requests)
  • Interacts with the Model to process data
  • Selects the appropriate View to render
  • Acts as the middleman between Model and View
  • The brain — decides what happens next
Analogy — Restaurant Order: You (the User) tell the waiter (the Controller) what you want. The waiter passes your order to the kitchen (the Model) which prepares the food from ingredients (the database). The waiter then brings the plated dish (the View) back to you. You never go into the kitchen directly, and the kitchen never comes to your table.

MVC Data Flow

User sends request (clicks link / submits form)
↓
Controller receives request, determines what action is needed
↓
Model performs database query, applies business rules, returns data
↓
Controller passes data to the appropriate View
↓
View renders HTML using the data from the Model
↓
Browser displays the HTML to the user

MVC Code Example

<?php
// === MODEL: User.php ===
class User {
    private $db;

    public function __construct($db) {
        $this->db = $db;
    }

    public function findAll() {
        $stmt = $this->db->query("SELECT * FROM users");
        return $stmt->fetchAll(PDO::FETCH_ASSOC);
    }

    public function findById($id) {
        $stmt = $this->db->prepare("SELECT * FROM users WHERE id = ?");
        $stmt->execute([$id]);
        return $stmt->fetch(PDO::FETCH_ASSOC);
    }
}

// === CONTROLLER: UserController.php ===
class UserController {
    private $userModel;

    public function __construct($db) {
        $this->userModel = new User($db);
    }

    public function index() {
        $users = $this->userModel->findAll();  // Ask Model for data
        require 'Views/user_list.php';          // Load the View
    }

    public function show($id) {
        $user = $this->userModel->findById($id);
        require 'Views/user_detail.php';
    }
}

// === VIEW: Views/user_list.php ===
// <h1>All Users</h1>
// <ul>
// <?php foreach ($users as $user): ?>
//   <li><?= htmlspecialchars($user['name']) ?></li>
// <?php endforeach; ?>
// </ul>
?>

8. MVC Benefits

BenefitExplanation
Separation of ConcernsEach component has a single, well-defined responsibility. Front-end developers work on Views, back-end developers work on Models.
Multiple RepresentationsSame data (Model) can be rendered as HTML, JSON, or XML — just swap the View.
Team CollaborationDesigners modify Views without touching logic. Developers modify Controllers/Models without breaking the layout.
Easier MaintenanceBug in data logic? Check the Model. Layout issue? Check the View. Input issue? Check the Controller.
ReusabilityModels can be reused across different Controllers. Views can be reused with different data.
ExtensibilityNew features can be added as new Models, Views, or Controllers without rewriting existing code.

9. PHP Frameworks — When to Use One

What is a PHP Framework?

A PHP framework is a reusable skeleton of libraries, tools, and coding conventions. It provides a structured foundation so you don't start from scratch.

Benefits of Using a Framework

  • Avoid reinventing the wheel — authentication, routing, database abstraction are already built
  • Enforce structure and standards — your code follows best practices by default
  • Speed up development — you write application logic, not boilerplate
  • Improve reliability — frameworks are battle-tested by thousands of developers
  • Maintenance benefits — follows conventions so other developers can understand your code

When to USE a Framework

  • The project will scale to many pages and features
  • You need long-term maintainability
  • A team is working on the project
  • You need built-in security features (CSRF, XSS protection, input sanitisation)

When NOT to Use a Framework

  • Building a very simple static site (a few HTML pages)
  • Creating a quick prototype or proof of concept
  • You don't want the overhead and learning curve
  • The project is a small script that won't grow
Warning: Using a framework for a simple 3-page website is like hiring a construction crew to build a birdhouse. The overhead outweighs the benefits. Choose the right tool for the job.

URL Routing with .htaccess

URL routing maps a user's request URL to the correct controller and method. Apache's .htaccess file rewrites all URLs to a single entry point:

# .htaccess - Redirect all requests to index.php
RewriteEngine On
RewriteCond %{REQUEST_FILENAME} !-f
RewriteCond %{REQUEST_FILENAME} !-d
RewriteRule ^(.*)$ index.php?url=$1 [QSA,L]
Analogy — Mail Sorting Office: The .htaccess is the mail sorting office. Every letter (URL request) that doesn't match a known address (existing file) gets redirected to the main office (index.php), which reads the address and routes it to the correct department (Controller).

The Front Controller (index.php)

The Front Controller is a single entry point that handles all requests. It parses the URL, loads the right controller, and calls the right method:

<?php
// index.php - Front Controller

// Get the URL parameter
$url = isset($_GET['url']) ? rtrim($_GET['url'], '/') : 'home';
$url = explode('/', $url);

// Determine controller and method
$controllerName = ucfirst(array_shift($url)) . 'Controller';
$methodName     = array_shift($url) ?: 'index';
$params         = $url;

// Load the controller file
$controllerFile = "controllers/$controllerName.php";
if (file_exists($controllerFile)) {
    require_once $controllerFile;
    $controller = new $controllerName();

    // Call the method with any remaining parameters
    if (method_exists($controller, $methodName)) {
        call_user_func_array([$controller, $methodName], $params);
    } else {
        echo "Method $methodName not found.";
    }
} else {
    echo "Controller $controllerName not found.";
}
?>

Core Classes

<?php
// core/Controller.php - Base Controller class
class Controller {
    protected $db;

    public function __construct() {
        $this->db = Database::getInstance()->getConnection();
    }

    protected function model($modelName) {
        $modelFile = "models/$modelName.php";
        require_once $modelFile;
        return new $modelName($this->db);
    }

    protected function view($viewPath, $data = []) {
        extract($data);
        $viewFile = "views/$viewPath.php";
        if (file_exists($viewFile)) {
            require_once $viewFile;
        }
    }
}

// core/Database.php - Database connection (Singleton)
class Database {
    private static $instance = null;
    private $connection;

    private function __construct() {
        $this->connection = new PDO(
            "mysql:host=localhost;dbname=mydb",
            "root", ""
        );
        $this->connection->setAttribute(
            PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION
        );
    }

    public static function getInstance() {
        if (self::$instance === null) {
            self::$instance = new self();
        }
        return self::$instance;
    }

    public function getConnection() {
        return $this->connection;
    }
}
?>
