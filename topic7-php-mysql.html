1. What is MySQL?

Definition

MySQL is an open-source Relational Database Management System (RDBMS) created in 1995 by Monty Widenius and David Axmark. It is the most popular open-source database used with PHP in the LAMP, WAMP, and XAMPP stacks.

Analogy — What is an RDBMS?
Imagine a spreadsheet program like Excel. Now imagine it is a full server application that many programs can talk to at the same time, that enforces rules about what data can go where, and that can handle millions of rows efficiently. That is essentially what an RDBMS is. MySQL is one specific RDBMS — like how "Ford" is a specific car brand. Other brands (RDBMS) include PostgreSQL, SQLite, and Oracle.

Real-World Usage

MySQL powers some of the biggest websites in the world:

  • Facebook — stores billions of user profiles, posts, and messages
  • WordPress — powers over 40% of all websites on the internet
  • Netflix — manages user accounts and viewing history
  • YouTube — handles video metadata and comments
  • Booking.com — manages hotel listings and reservations
  • Airbnb — stores listings, reviews, and booking data
Note: MySQL is part of the LAMP stack (Linux, Apache, MySQL, PHP) — the classic combination of technologies that powers most web applications. When you install XAMPP or WAMP on your computer, MySQL comes bundled and ready to use.

What does RDBMS stand for?

Relational Database Management System — a system that stores data in tables with defined relationships between them.

Who created MySQL and when?

Monty Widenius and David Axmark in 1995.

Name 3 companies that use MySQL.

Facebook, WordPress, Netflix (also YouTube, Booking.com, Airbnb).

2. phpMyAdmin

What is phpMyAdmin?

phpMyAdmin is a free, web-based tool for administering MySQL databases. It has been around since 1998 and is written entirely in PHP. It provides a graphical interface so you don't have to type every SQL command by hand.

Analogy: If MySQL is the pantry, then phpMyAdmin is the glass window into the pantry that lets you see all the shelves, move items around, and reorganize — all without needing to know the exact language the pantry shelf-labels are written in. You can use SQL directly, but phpMyAdmin also gives you buttons, forms, and visual tools.

3. What is MariaDB?

The Story of MariaDB

MariaDB is a fork (a copy that diverges into its own project) of MySQL. In 2009, Oracle Corporation acquired Sun Microsystems (which owned MySQL). Many in the open-source community were concerned about the future of MySQL under Oracle, so Monty Widenius (one of MySQL's original creators) forked the codebase and created MariaDB.

Analogy: Imagine a popular recipe that belongs to a restaurant. When the restaurant is bought by a large corporation, the original chef is worried the recipe might change or become restricted. So the chef takes the recipe, opens a new restaurant, and keeps cooking the same dish — maybe even improving it. The food tastes the same (compatible), but now it is under new management. MariaDB is the chef's new restaurant.

Why MariaDB?

  • Full binary "drop-in" replacement — you can swap MySQL for MariaDB and everything just works
  • Compatible data — same database files, same formats
  • Compatible APIs and protocols — same code talks to both
  • Same filenames and ports — same default port 3306
  • Maintains MySQL connectors — PDO, mysqli, and other drivers work with MariaDB too
Note: In practice, for this course it does not matter whether you use MySQL or MariaDB. The SQL syntax, PHP connection code, and tools like phpMyAdmin work identically with both.

4. Database Queries & CRUD

What is a Query?

A database query is a request to access, retrieve, modify, or delete data in a database. Every time your PHP script needs information from the database, it sends a query.

Analogy — CRUD: Think of a library system. Create = adding a new book to the catalogue. Read = searching for and reading a book's details. Update = correcting a typo on a book's info card. Delete = removing a book that has been discarded. Every database operation falls into one of these four categories.

The Four CRUD Operations

CRUDSQL StatementWhat It DoesExample
Create INSERT Adds a new row to a table INSERT INTO users (name, email) VALUES ('Ali', 'ali@mail.com');
Read SELECT Retrieves data from a table SELECT * FROM users WHERE id = 1;
Update UPDATE Modifies existing rows UPDATE users SET email = 'new@mail.com' WHERE id = 1;
Delete DELETE Removes rows from a table DELETE FROM users WHERE id = 1;
Warning: Always be careful with DELETE and UPDATE statements. Without a WHERE clause, they will affect every row in the table. Imagine accidentally deleting every user in your system!

Which SQL statement corresponds to the "Read" operation in CRUD?

SELECT

What does INSERT do?

It adds a new row (record) to a database table — the "Create" in CRUD.

What happens if you run UPDATE without a WHERE clause?

Every row in the table gets updated — usually a disaster!

7. Connecting to MySQL — Step by Step

Step 1: Create a PDO instance with your credentials
Step 2: Set the error mode to ERRMODE_EXCEPTION
Step 3: Wrap everything in a try/catch block for exception handling
Step 4: Close the connection (set variable to null)

Full Connection Example

<?php
$host = '127.0.0.1';
$db   = 'my_database';
$user = 'root';
$pass = '';
$charset = 'utf8mb4';

$dsn = "mysql:host=$host;dbname=$db;charset=$charset";
$options = [
    PDO::ATTR_ERRMODE            => PDO::ERRMODE_EXCEPTION,
    PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC,
    PDO::ATTR_EMULATE_PREPARES   => false,
];

try {
    $pdo = new PDO($dsn, $user, $pass, $options);
    echo "Connected successfully!";
} catch (\PDOException $e) {
    throw new \PDOException($e->getMessage(), (int)$e->getCode());
}

// When done, close connection:
$pdo = null;
?>
Tip — The DSN: DSN stands for Data Source Name. It is a string that tells PDO which database driver to use, where the server is, and which database to connect to. Format: "mysql:host=HOSTNAME;dbname=DATABASE;charset=CHARSET"
Security: Never hard-code your database password in a production file that could be exposed. Use environment variables or a separate config file outside the web root.

8. Running Queries — query() vs exec()

Two Methods for Two Purposes

MethodUse ForReturns
$pdo->query($sql)SELECT statementsA PDOStatement object (result set you can loop through)
$pdo->exec($sql)INSERT, UPDATE, DELETEThe number of rows affected (an integer)

Example: SELECT with query()

<?php
$stmt = $pdo->query('SELECT name, email FROM users');
while ($row = $stmt->fetch()) {
    echo $row['name'] . ' - ' . $row['email'] . '<br>';
}
?>

Example: INSERT with exec()

<?php
$count = $pdo->exec("INSERT INTO users (name, email) VALUES ('Siti', 'siti@mail.com')");
echo "Rows inserted: $count";
?>

SQL Rules Inside PHP

  • The entire SQL query must be quoted as a PHP string (single or double quotes)
  • String values in SQL must be quoted: VALUES ('Ali')
  • Numeric values do NOT need quotes: WHERE id = 1
  • NULL values do NOT get quotes: VALUES (NULL)

Which method returns a PDOStatement — query() or exec()?

query() returns a PDOStatement. exec() returns the affected row count.

9. Getting the Last Inserted Record

lastInsertId()

When you insert a row into a table with an AUTO_INCREMENT primary key, MySQL automatically assigns the next number. PHP can retrieve this number using $pdo->lastInsertId().

Analogy: Imagine you are at a ticket counter. You fill out a form and hand it in. The clerk stamps it with ticket number 42. You did not choose 42 — the system assigned it. lastInsertId() is like asking "what number did the system just give me?" and the clerk answers "42."

Example

<?php
$pdo->exec("INSERT INTO users (name, email) VALUES ('Ahmed', 'ahmed@mail.com')");
$newId = $pdo->lastInsertId();
echo "New user created with ID: $newId";
?>
Note: lastInsertId() only works with AUTO_INCREMENT columns. If your table does not use AUTO_INCREMENT, this method will not return a meaningful value.

10. Transactions

What is a Transaction?

A transaction is a group of database operations that are treated as a single unit. Either all of them succeed, or none of them are applied. This ensures data integrity.

Analogy — The Bank Transfer: You transfer RM 500 from Account A to Account B. This involves two steps: (1) subtract RM 500 from A, (2) add RM 500 to B. What if step 1 succeeds but step 2 fails? You'd lose RM 500! A transaction ensures that if step 2 fails, step 1 is also undone (rolled back). It is all-or-nothing.
beginTransaction() — marks the start of a transaction. Returns true on success.
commit() — saves all changes made since beginTransaction(). Returns true on success.
rollBack() — undoes all changes made since beginTransaction(). Returns true on success.

Example

<?php
try {
    $pdo->beginTransaction();

    $pdo->exec("UPDATE accounts SET balance = balance - 500 WHERE id = 1");
    $pdo->exec("UPDATE accounts SET balance = balance + 500 WHERE id = 2");

    $pdo->commit();
    echo "Transfer successful!";
} catch (Exception $e) {
    $pdo->rollBack();
    echo "Transfer failed: " . $e->getMessage();
}
?>
Warning: Not all database tables support transactions. The table must use a transactional engine like InnoDB (the default in modern MySQL). The older MyISAM engine does not support transactions.

11. Fetching Data — fetch() vs fetchAll()

Two Ways to Retrieve Results

A PDOStatement can only be looped through once. Once you have iterated over it, you cannot go back to the beginning. This is because it is a forward-only cursor — like reading a scroll from top to bottom.

Analogy: Imagine a slot machine. fetch() is like pulling the lever once — you get one result. fetchAll() is like asking the machine to dump all its results into a bucket at once.
MethodWhat It DoesReturns
fetch()Retrieves the next single row from the result setOne row as an associative array, or false when no more rows
fetchAll()Retrieves all remaining rows at onceA nested array (all rows)

Using fetch() — One Row at a Time

<?php
$stmt = $pdo->query('SELECT * FROM users');
while ($row = $stmt->fetch()) {
    echo $row['name'] . ' — ' . $row['email'] . '<br>';
}
?>

Using fetchAll() — All Rows at Once

<?php
$stmt = $pdo->query('SELECT * FROM users');
$users = $stmt->fetchAll();

foreach ($users as $user) {
    echo $user['name'] . ' — ' . $user['email'] . '<br>';
}
?>
Tip: Use fetch() when you have many rows and want to process them one at a time (memory efficient). Use fetchAll() when you need all the data at once (convenient for small datasets or when you want to use array functions).

Can a PDOStatement be looped more than once?

No. A PDOStatement is forward-only and can only be iterated once. If you need the data again, use fetchAll() to store it in an array first.

12. Prepared Statements & SQL Injection

What are Prepared Statements?

A prepared statement is a two-step process: first you create a SQL template with placeholders, then you execute it by binding actual values. This separates the SQL logic from the data.

Analogy — Mad Libs: You know the word game Mad Libs? You create a story template with blanks: "The ___ (adjective) cat sat on the ___ (noun)." Later, someone fills in the blanks. The story structure is decided before you know the words. Prepared statements work the same way — the SQL structure is decided before the data is plugged in. This prevents the data from "changing the story."

Three Advantages of Prepared Statements

  • Reduced parsing time — the database prepares the query plan once and can reuse it
  • Minimized network bandwidth — the template is sent once, only values are sent on each execution
  • Prevents SQL Injection — user input is never treated as SQL code, only as data

SQL Injection Attack

SQL Injection is a security vulnerability where an attacker inserts malicious SQL code into an input field. Consider this scenario:

Dangerous Example (DO NOT do this):

A login form checks:
$sql = "SELECT * FROM login WHERE username='$user' AND password='$pass'";

An attacker enters this password:
x'; DELETE FROM login WHERE '0'='0

The resulting SQL becomes:
SELECT * FROM login WHERE username='attacker' AND password='x'; DELETE FROM login WHERE '0'='0';

This would first check the login, then delete every single row in the login table!

Prepared Statement — Preventing SQL Injection

<?php
// STEP 1: Prepare the template with placeholders
$stmt = $pdo->prepare('SELECT * FROM users WHERE email = :email AND password = :pass');

// STEP 2: Execute with actual values
$stmt->execute([
    ':email' => $email,
    ':pass'  => $password
]);

$user = $stmt->fetch();
?>

Alternative: bindParam()

<?php
$stmt = $pdo->prepare('INSERT INTO users (name, email) VALUES (:name, :email)');
$stmt->bindParam(':name', $name);
$stmt->bindParam(':email', $email);

$name  = 'Siti';
$email = 'siti@mail.com';
$stmt->execute();
?>
Note: There are two types of placeholders: named (:email) and positional (?). Named placeholders are recommended because they are more readable and less error-prone.

What is SQL Injection?

An attack where malicious SQL code is inserted into input fields to manipulate the database query, potentially destroying or stealing data.

How do prepared statements prevent SQL injection?

They separate the SQL template (structure) from the data (values). User input is always treated as literal data, never as executable SQL code.

What are the 3 advantages of prepared statements?

1. Reduced parsing time (query plan reused). 2. Minimized bandwidth (template sent once). 3. Prevents SQL injection.
