Welcome to your first topic! In this lesson we will understand what the course is about, explore the concept of a technology stack, and learn the roles of frontend and backend in web development.

1. What is TP2543?

TP2543 is a course focused on Web Application Development. You will learn how to build complete web applications from scratch, covering both the visual side that users see and interact with, and the server side that processes data behind the scenes.

🏪 Analogy — Building a House: Think of TP2543 as learning to build an entire house. Both parts are essential for a livable house, just as both frontend and backend are essential for a web application.

2. What is a Technology Stack?

A technology stack (also called a solution stack or software stack) is a combination of software technologies, programming languages, frameworks, and third-party tools that work together to build and run a web application. Selecting the right combination of technologies is a critical and challenging decision in any web development project.

🏪 Analogy — A Restaurant:
  • Frontend = The Dining Room — This is what customers see: the tables, chairs, menus, decor, lighting, and the waiters who take their orders. It’s the entire experience the customer interacts with.
  • Backend = The Kitchen — This is where the actual work happens. Chefs (server logic) prepare the food (process data), using ingredients from the pantry (database). Customers never see the kitchen, but without it, there is no meal.
  • Technology Stack = The Entire Restaurant Setup — The ovens, fridges, utensils, menu design, table layout, ordering system — everything that works together to deliver a meal to the customer.
💡 Key Insight: Popular technology stacks exist because certain combinations of tools have proven to work well together. Examples include LAMP (Linux, Apache, MySQL, PHP), MERN (MongoDB, Express, React, Node.js), and MEAN (MongoDB, Express, Angular, Node.js).

3. Frontend (Client-Side)

The frontend is the visual side of a web application — everything that a user sees and interacts with in their web browser. It is sometimes called the client-side because it runs on the client’s device (your computer, phone, or tablet).

The frontend is built using three core technologies:

  • HTML (HyperText Markup Language) — Provides the structure and content of web pages (headings, paragraphs, images, links).
  • CSS (Cascading Style Sheets) — Controls the visual presentation and styling (colors, fonts, layout, spacing).
  • JavaScript — Adds interactivity and dynamic behavior (button clicks, animations, form validation, updates without page reload).

When you type a URL into your browser and see a webpage, the browser is executing frontend code to render the page on your screen.

🏪 Analogy — The Dining Room: The frontend is like the dining room of a restaurant. The layout of tables, the design of the menu, the color of the walls, and the way the waiter presents the food — all of this creates the dining experience. If the dining room is messy, poorly lit, or confusing, customers will have a bad experience even if the food (backend) is excellent.
▶ Deeper Dive: The Three Pillars of Frontend

HTML — The Structure

HTML is not a programming language; it is a markup language. It uses tags (like <h1>, <p>, <div>) to define the structure of content. Think of HTML as the 骨架 (skeleton) of a webpage — it defines what elements exist but not how they look.

CSS — The Style

CSS is a Style Sheet Language. It tells the browser how to visually present HTML elements. Using selectors and properties, CSS controls colors, fonts, sizes, positioning, animations, and responsive layouts.

JavaScript — The Behavior

JavaScript is a true programming/scripting language. It adds logic, interactivity, and dynamic content. JavaScript can respond to user events (clicks, typing), modify HTML and CSS on the fly, make network requests, and much more.

💡 Tip: In this course, you will study all three extensively. We start with HTML, move to CSS, and then add JavaScript.

4. Backend (Server-Side)

The backend is the server-side of a web application. It handles data processing, business logic, database interactions, user authentication, and more. The backend is not visible to the user — it works behind the scenes.

The backend typically consists of several software components:

  • HTTP Server (Web Server) — Software that receives requests from the browser and sends back responses (e.g., Apache, Nginx). It serves static files (HTML, CSS, images) and forwards dynamic requests to the application server.
  • Application Server — Software that executes the business logic of your application. This is where server-side programming languages like PHP, Python, Java, or Node.js run.
  • Database Server — Software that stores, retrieves, and manages data (e.g., MySQL, PostgreSQL, MongoDB). When your application needs to read, add, edit, or delete data, it communicates with the database server.

All these components together form the business logic of the product.

🏪 Analogy — The Kitchen:
  • Web Server = The Order Window — The place where orders (requests) come in from the dining room and prepared dishes (responses) go back out.
  • Application Server = The Chefs — They read the order, decide what to cook, follow recipes (business logic), and prepare the dish.
  • Database Server = The Pantry — Where all the ingredients (data) are stored. The chefs reach in to get what they need.
▶ Deeper Dive: Backend Components Explained

HTTP Server (Web Server)

An HTTP server understands URLs and HTTP protocol. When you type http://example.com/page.php, the HTTP server receives this request. If the file is static (like an image), it sends it directly. If it is dynamic (like a PHP script), it passes the request to the application server.

Application Server

The application server runs your server-side code. In this course, we use PHP. When a PHP script is requested, the application server (e.g., Apache with PHP module in XAMPP) executes the PHP code, which may connect to the database, process data, and generate HTML output to send back to the browser.

Database Server

A database server manages structured data using SQL (Structured Query Language) in the case of relational databases like MySQL. Your PHP code will use SQL to perform CRUD operations: Create, Read, Update, Delete data.

⚠️ Warning: Understanding the backend is crucial because in this course you will build the server-side logic using PHP and MySQL. Do not skip backend topics!

5. How the Technology Stack Parts Interact

The frontend and backend do not exist in isolation. They communicate through HTTP requests and responses. Here is the flow:

💻 CLIENT (Browser)
HTML + CSS + JavaScript
↓ 📨 HTTP Request ↓
☁️ WEB SERVER (Apache / HTTP Server)
Serves files & forwards dynamic requests
↓ 🔄 Process & Execute ↓
⚙️ APPLICATION SERVER (PHP / Server-Side Logic)
Business logic, data processing
↓ 🔍 SQL Query ↓
🗄️ DATABASE SERVER (MySQL)
Stores & retrieves data

The interaction happens in three color-coded scenarios:

  1. Blue — Simple Request: The browser requests a simple open HTML file or graphic file (image, CSS, JS). The web server serves the file directly — no processing needed.
  2. Green — Complex Request: The browser requests something that requires processing, like running a PHP server-side script. The web server passes the request to the application server, which executes the script and returns the result.
  3. Yellow — Data Request: The browser requests to read, add, edit, or remove data (CRUD operations). The server-side script makes a SQL request to the database server, receives the data, processes it, and sends the result back to the client.
🏪 Analogy — Ordering Food:
  • Blue (Simple): Customer asks for a glass of water. The waiter brings it straight from the tap. No cooking needed.
  • Green (Complex): Customer orders a dish that requires preparation. The waiter takes the order to the kitchen, a chef cooks it, and it comes back ready.
  • Yellow (Data): Customer asks “Do you have today’s specials?” The waiter checks with the kitchen, who checks the pantry inventory, and reports back what’s available.

CRUD
An acronym for Create, Read, Update, Delete — the four basic operations performed on data in a database.

9. Practical Tips for Success

▶ Tutorial 1: Setup XAMPP and Sublime Editor

Before your first lab, you need to set up your development environment:

  1. Download and install XAMPP — XAMPP includes Apache (web server + application server) and MySQL (database server). It’s a free, easy-to-use package that gives you a complete backend environment on your local machine (localhost).
  2. Download and install Sublime Text — Sublime Text is a code editor where you will write your HTML, CSS, PHP, and other code files.
  3. Start Apache and MySQL in the XAMPP Control Panel. Your web server and database server will now be running locally.
  4. Test: Open your browser and go to http://localhost. If you see the XAMPP welcome page, your setup is working.
▶ Tutorial 2: Set Up Your Server Account

In addition to localhost, you will set up a server account for deploying your projects. Follow the instructions given by your lecturer to create and configure your server account. This account will be used to host your web application so it can be accessed over the network.

▶ General Tips for This Course
  • All labs are related to each other — Each lab builds on the previous one. Missing one lab can create a chain of confusion.
  • Do not procrastinate — Start early, work consistently. Web development requires hands-on practice.
  • Practice, practice, practice — Reading is not enough. Write code, experiment, make mistakes, and learn from them.
  • Learn from real examples at w3schools.com — A fantastic resource with interactive examples for HTML, CSS, JavaScript, PHP, SQL, and more.
