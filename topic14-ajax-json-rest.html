Topic 14: AJAX, JSON & RESTful APIs

1. The Problem: Why AJAX?

Analogy — The Library vs. The Internet: In the old days, every time you had a question, you had to walk to the library, check out an entire book, bring it home, read one paragraph, and then return the whole book before checking out another. That's how traditional web pages worked — you loaded the entire page just to see one small change. AJAX is like having a personal assistant who runs to the library, finds the exact paragraph, and brings it back to you — without you ever leaving your chair.

Before AJAX, web applications had a usability problem: every interaction with the server required a full page reload. This made web apps feel sluggish compared to desktop applications.

The Goal: Rich Internet Applications (RIAs)

RIAs aim to deliver the same look, feel, and usability as desktop applications inside the browser. Two key attributes:

  1. High performance — instant feedback, no unnecessary waiting
  2. Rich GUI — responsive, interactive user interfaces

RIAs achieve performance through AJAX, which uses client-side scripting to create responsive applications by separating user interaction from server communication.

Everyday AJAX Examples

  • News feeds that auto-update without refreshing the page
  • Like buttons that update the count instantly
  • Google search suggestions that appear as you type
  • Social media timelines that load more posts when you scroll down

2. Synchronous vs. Asynchronous

Analogy — The Single Cashier vs. The Self-Checkout:

Synchronous: A store with one cashier and a long line. Every customer must wait for the person in front to be fully served before their turn. The cashier cannot multitask.

Asynchronous: A self-checkout area with multiple machines. Each customer scans their items independently. While one customer is bagging, another is scanning. No one blocks anyone else.

Synchronous Processing

  • One thing happens after another — strictly sequential
  • The server cannot multitask for the client
  • You cannot interact with the page while waiting for a response
  • The entire page freezes until the request completes

Asynchronous Processing

  • The page continues processing while waiting for a response
  • The reply is handled when it arrives (via a callback)
  • No full page reload is needed — only the affected part updates
  • The user can keep interacting with the page
Synchronous
1. User clicks button
2. Browser sends request
3. Page is frozen ⏳
4. Server responds
5. Entire page reloads
6. User can interact
Asynchronous
1. User clicks button
2. Browser sends request
3. Page keeps working 🎉
4. Server responds
5. Only the changed part updates
6. User never stopped interacting
Loading indicators (spinners, progress bars) are commonly shown during asynchronous requests so users know something is happening in the background.

3. What is AJAX?

Analogy — The Messenger: AJAX is like a messenger you send out to deliver a letter and bring back a reply. While the messenger is walking to the post office, you can keep doing other things — cooking, reading, talking. When the messenger returns, you read the reply and act on it. The key point: you didn't have to stop everything and walk to the post office yourself.

AJAX stands for Asynchronous JavaScript and XML. It is a set of web development techniques that use client-side technologies to create asynchronous web applications.

What AJAX Does

  • Sends data to a server in the background
  • Receives data from a server
  • Updates the page without a full reload
  • All done without interfering with the current display

Two Ways to Implement AJAX

MethodDescription
Raw JavaScriptUsing the XMLHttpRequest object directly — verbose but fundamental
jQueryUsing jQuery's AJAX methods — simpler, cross-browser compatible

4. XMLHttpRequest — The Raw Approach

Analogy — Writing a Formal Letter: Using XMLHttpRequest directly is like writing a formal letter by hand: you address the envelope, write the content, seal it, and send it. It works, but it's a lot of steps. (jQuery's AJAX methods, covered next, are like sending an email — same result, much less effort.)

XMLHttpRequest is a built-in browser object that allows you to send HTTP requests from JavaScript.

Basic Usage

// 1. Create the request object
var request = new XMLHttpRequest();

// 2. Configure the request (method, URL)
request.open("GET", "about_us.txt");

// 3. Define what happens when the response arrives
request.onreadystatechange = function() {
    if (request.readyState === 4 && request.status === 200) {
        document.getElementById("output").innerHTML = request.responseText;
    }
};

// 4. Send the request
request.send();

The open() Method

ExampleHTTP MethodPurpose
request.open("GET", "about_us.txt")GETRetrieve data
request.open("POST", "process.php")POSTSend data

The send() Method

UsageWhen
request.send()For GET requests (no body needed)
request.send(body)For POST requests (sends data to the server)

readyState Values

The readyState property tracks the progress of the request through these stages:

ValueConstantMeaning
0UNSENTRequest not yet opened
1OPENEDopen() has been called
2HEADERS_RECEIVEDResponse headers received
3LOADINGReceiving response data
4DONERequest complete, response ready

HTTP Status Codes

CodeMeaning
200OK — request succeeded
404Not Found — resource doesn't exist
503Service Unavailable — server can't handle the request
Key Check: Always check both readyState === 4 (request finished) AND status === 200 (successful) before processing the response.

5. jQuery AJAX Methods

Analogy — Speaking the Language: If raw XMLHttpRequest is speaking to the server in its native protocol, jQuery AJAX methods are your translator who makes it easy and natural. Same message, but much more pleasant to use.

jQuery provides several convenient methods for AJAX requests. All of them request text, HTML, XML, or JSON from a remote server using HTTP GET or POST.

Overview of jQuery AJAX Methods

MethodPurpose
.load()Load data from a server into an HTML element
$.get()Simple GET request
$.post()Simple POST request
$.ajax()Full-featured, configurable AJAX requests
Same-Origin Policy: By default, all AJAX requests can only communicate with the same domain/server the page was loaded from. This is a critical security restriction.

6. The .load() Method

.load() is the simplest jQuery AJAX method. It fetches HTML from a server and inserts it directly into a selected element.

Syntax

$(selector).load(URL, data, callback);
  • URL — The address to fetch data from
  • data (optional) — Data to send with the request
  • callback (optional) — Function that runs after loading

Basic Example

// Load content from "about.html" into #mainContent
$("#mainContent").load("about.html");

With Callback

$("#result").load("data.html", function(responseTxt, statusTxt, xhr) {
    if (statusTxt === "success") {
        console.log("Content loaded successfully!");
    }
    if (statusTxt === "error") {
        console.log("Error: " + xhr.status + ": " + xhr.statusText);
    }
});

Load a Specific Portion

You can fetch only a part of a page by adding a CSS selector after a space:

// Only load elements with class "summary" from news.html
$("#newsArea").load("news.html .summary");

Callback Parameters

ParameterDescription
responseTxtThe content returned from the server
statusTxt"success" or "error"
xhrThe XMLHttpRequest object (for detailed status info)

7. $.get() and $.post()

Analogy — Asking vs. Telling: $.get() is like asking a question — "Hey server, give me this information." $.post() is like sending a package — "Hey server, here's some data, please process it." Both reach the same server, but the way they send information differs.

$.get() — Simple GET Request

$.get("api/users", function(data, status) {
    alert("Data: " + data + "\nStatus: " + status);
});

$.post() — Simple POST Request

$.post("api/users", {
    name: "John Doe",
    email: "john@example.com"
}, function(data, status) {
    alert("Response: " + data + "\nStatus: " + status);
});

Comparison

HTTP MethodGETPOST
Data sent viaURL query stringRequest body
Use caseRetrieving dataSubmitting data
CacheableYesNo
Data limitLimited by URL lengthNo practical limit
Important: Both $.get() and $.post() can only communicate with the same domain the page was loaded from.

8. $.ajax() — The Power Tool

Analogy — The Swiss Army Knife: While .load(), $.get(), and $.post() are each one specific tool, $.ajax() is the full Swiss Army knife — it can do everything they do and much more. You configure it with an options object that specifies every detail of the request.

Syntax

$.ajax({
    type: "POST",              // HTTP method
    url: "api/register",       // Destination URL
    data: {                    // Data to send
        name: "John",
        age: 25
    },
    dataType: "json",          // Expected response type
    beforeSend: function() {   // Runs before sending
        $("#loading").show();
    },
    success: function(response) {  // Runs on success
        console.log("Success:", response);
        $("#result").html(response.message);
    },
    error: function(xhr, status, error) {  // Runs on failure
        console.error("Error:", error);
        $("#result").html("Something went wrong.");
    },
    complete: function() {     // Runs after success or error
        $("#loading").hide();
    }
});

Key Configuration Options

OptionDescription
typeHTTP method: "GET", "POST", "PUT", "DELETE"
urlThe URL to send the request to
dataData to send (object, query string, or FormData)
dataTypeExpected response: "json", "html", "text", "xml"
beforeSendFunction to run before the request is sent
successFunction to run when the request succeeds
errorFunction to run when the request fails
completeFunction to run after success or error
Why $.ajax() is preferred for complex scenarios: It gives you full control over every aspect of the request — headers, timeouts, error handling, data types, and lifecycle callbacks.

9. JSON — JavaScript Object Notation

Analogy — The Universal Language of Data: Imagine two people who speak different languages trying to exchange information. They agree on a shared notation system using simple symbols everyone understands. JSON is that system for computers. It's a lightweight, text-based format that every programming language can read and write — like a lingua franca for data.

JSON (JavaScript Object Notation) is a lightweight, text-based, open standard for human-readable data interchange.

Key Facts

  • Extended from JavaScript object literal syntax
  • Files use the .json extension
  • Media type: application/json
  • Language independent — while it originated from JavaScript, every major language can parse and generate JSON

JSON Syntax

{
    "name": "value"
}

Example: A Person Object

{
    "first": "John",
    "last": "Doe",
    "age": 20,
    "phone": null
}

Arrays of Objects

[
    {
        "first": "John",
        "last": "Doe",
        "age": 20
    },
    {
        "first": "Jane",
        "last": "Smith",
        "age": 22
    }
]

Valid JSON Value Types

TypeExample
String"Hello"
Number42, 3.14
Object{"key": "value"}
Array[1, 2, 3]
Booleantrue, false
Nullnull
Invalid JSON types: JSON does NOT support functions, dates, or undefined. These must be converted to strings or numbers before sending.

Parsing and Encoding

OperationJavaScriptPHP
Parse (string → object)JSON.parse()json_decode()
Encode (object → string)JSON.stringify()json_encode()

JavaScript Examples

// Parsing JSON string into a JavaScript object
var jsonString = '{"first":"John","last":"Doe","age":20}';
var obj = JSON.parse(jsonString);
console.log(obj.first); // "John"

// Encoding JavaScript object into a JSON string
var person = { first: "Jane", last: "Smith", age: 22 };
var jsonString = JSON.stringify(person);
console.log(jsonString); // '{"first":"Jane","last":"Smith","age":22}'

10. Web Services

Analogy — The International Postal System: A web service is like the global postal system. You (the front-end) write a letter (request) in a standard format (HTTP). The post office (the Internet) delivers it to any address (server) in the world. The server writes back in the same standard format. Neither side needs to know how the other works internally — they just follow the postal rules (protocols).

A web service is a collection of open protocols and standards used for exchanging data between systems over the Internet.

The Architecture

  • Front-end (your browser/device) sends requests
  • Data lives on a remote server
  • APIs (Application Programming Interfaces) are the gateways
  • Data is transferred via HTTP in formats like XML or JSON

Two Major Architectures

ArchitectureDescription
SOAPSimple Object Access Protocol — XML-based, strict, enterprise-focused
RESTRepresentational State Transfer — lightweight, flexible, modern

11. RESTful Web Services

Analogy — A Restaurant Menu: A REST API is like a well-organized restaurant menu. Each dish has a clear name (URI), and the menu tells you exactly how to order it. Want the grilled chicken? That's a GET request to /api/chicken. Want to create a custom order? That's a POST to /api/orders. The kitchen (server) prepares it and sends it back. You don't need to know how they cook it — you just follow the menu.

REST (Representational State Transfer) is an architectural style that uses HTTP methods for CRUD operations on resources.

CRUD = HTTP Methods

CRUD OperationHTTP MethodDescription
CreatePOSTCreate a new resource
RetrieveGETRead/fetch resource(s)
UpdatePUTUpdate an existing resource
DeleteDELETERemove a resource

REST URI Convention — Employee Example

URIMethodAction
/api/employeesGETList all employees
/api/employees/1GETGet employee with ID 1
/api/employeesPOSTCreate a new employee
/api/employees/1PUTUpdate employee with ID 1
/api/employees/1DELETEDelete employee with ID 1
Design Principle: REST URIs should be intuitively understandable. The URI describes the resource (/api/employees), and the HTTP method describes the action.

12. API Testing

API testing validates APIs for:

  • Functionality — Does it do what it should?
  • Reliability — Is it consistent across requests?
  • Performance — Does it respond quickly under load?
  • Security — Is it protected against unauthorized access?

Postman — The API Client

Analogy — The Walkie-Talkie: Postman is like a walkie-talkie designed specifically for talking to APIs. You can type your message (request), choose the channel (HTTP method), set the frequency (headers), and hear the response — all without writing a single line of code.
  • Used for creating, sharing, testing, and documenting APIs
  • Supports all HTTP methods (GET, POST, PUT, DELETE, etc.)
  • Can set headers, body, authentication, and parameters
  • Displays response status, body, headers, and timing

13. Single-Page Applications (SPA)

Analogy — The Magic Book: Imagine a book where, instead of turning to a completely new page, the text on the current page magically rewrites itself to show new content. The cover, the page numbers, and your bookmarks stay the same — only the content changes. That's a SPA: one HTML page that dynamically updates its content without ever loading a new page.

A Single-Page Application dynamically rewrites the current page with new data instead of loading entire new pages from the server.

How It Works

  1. The browser loads one single HTML page initially
  2. When the user interacts, AJAX requests fetch data from the server
  3. The server responds with JSON data (not a full HTML page)
  4. JavaScript updates the DOM with the new data
  5. The URL may update via the History API (for bookmarkability)

Popular SPA Examples

ApplicationWhat It Does
GmailEmail client that never reloads — tabs, compose, inbox all update dynamically
Google MapsPans, zooms, and loads map tiles without page refresh
AirBNBSearch results, filters, and listings update seamlessly
NetflixBrowse categories, watch previews, all without full reloads

14. Putting It All Together

Complete Example: AJAX + jQuery + REST + JSON

<!DOCTYPE html>
<html>
<head>
  <script src="jquery-3.6.0.min.js"></script>
</head>
<body>
  <button id="loadBtn">Load Employees</button>
  <div id="result"></div>

  <script>
    $(function() {

      $("#loadBtn").on("click", function() {
        // Show loading indicator
        $("#result").html("<p>Loading...</p>");

        // GET request to REST API
        $.ajax({
          type: "GET",
          url: "/api/employees",
          dataType: "json",
          success: function(data) {
            // data is already parsed from JSON
            var html = "<table><tr><th>Name</th><th>Email</th></tr>";
            data.forEach(function(emp) {
              html += "<tr><td>" + emp.name + "</td>";
              html += "<td>" + emp.email + "</td></tr>";
            });
            html += "</table>";
            $("#result").html(html);
          },
          error: function(xhr, status, error) {
            $("#result").html("<p>Error loading data.</p>");
          }
        });
      });

    });
  </script>
</body>
</html>
This example demonstrates: Document ready, event handling, $.ajax() with configuration, GET request to a REST endpoint, JSON response handling, dynamic DOM update, and error handling — the complete modern web development cycle.

Summary

  • AJAX enables asynchronous communication with the server, eliminating full page reloads and creating responsive RIAs.
  • Synchronous blocks the page; Asynchronous lets the page continue while waiting for responses.
  • XMLHttpRequest is the built-in browser object for AJAX — check readyState === 4 and status === 200.
  • jQuery AJAX methods: .load() (simple HTML loading), $.get() (GET), $.post() (POST), $.ajax() (full control).
  • JSON is a lightweight data format supporting strings, numbers, objects, arrays, booleans, and null — not functions or dates.
  • Parse JSON: JSON.parse(). Encode JSON: JSON.stringify().
  • REST uses HTTP methods for CRUD: POST (Create), GET (Read), PUT (Update), DELETE (Delete).
  • REST URIs are resource-based: /api/employees + HTTP method = complete operation.
  • Postman is used for testing, sharing, and documenting APIs.
  • SPAs (like Gmail, Netflix) use AJAX + JSON to dynamically update a single HTML page.
