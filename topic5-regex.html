Topic 5: Regular Expressions (Regex)

Analogy — Regex is like a Stamp: Imagine you have a rubber stamp with a specific shape cut out. You drag that stamp across a page of text, and it only "inks" the parts that match the shape. A regex pattern is your stamp — it defines a shape of text, and the regex engine drags it across your string to find matches. Simple stamp = simple match. Complex stamp = complex pattern matching.

1. What Are Regular Expressions?

▼

Regular Expressions (commonly called regex or RegExp) are specially formatted text strings used to find patterns in other text. They are one of the most powerful tools available for text processing and manipulation.

Analogy — Regex as a Detective: Imagine you are a detective looking for a suspect who matches a description: "tall, wearing a red hat, carrying a briefcase." You don't know the suspect's name, but you know these characteristics. A regex pattern works the same way — it describes the characteristics of the text you're looking for, not the exact text itself. You give the regex engine a "description" and it finds all matching "suspects" in your text.

What Can Regex Do?

Regex is used for two main purposes:

  1. Verify data format — Is this a valid email address? Does this phone number follow the right pattern? Is this matric number properly formatted?
  2. Find and/or replace matching strings — Find all phone numbers in a document, replace all dates from one format to another, extract all links from a webpage.
Where is regex used? Regex is used everywhere — in programming languages (PHP, JavaScript, Python, Java), in text editors (VS Code, Sublime Text), in command-line tools (grep, sed), in form validation, in search-and-replace operations, and much more. Learning regex once gives you a skill that transfers across all these tools.

A Simple Example

Suppose you want to check if a string contains the word "cat":

Pattern: cat
Matches: "cat" in "The cat sat on the mat"

Simple enough. But what if you want to find any three-letter word starting with "c" and ending with "t"? That's where regex special syntax comes in.

Pattern: c.t (the dot matches any single character)
Matches: "cat", "cot", "cut", "cit" in "The cat sat on the cot"

2. Regex Syntax & Special Characters

▼

Regex uses a set of special characters that have specific meanings. These characters are the building blocks of every regex pattern.

Analogy — Special Characters are Like Instructions: Think of regex as a recipe. The special characters are like cooking instructions: "chop" means cut into pieces, "simmer" means cook slowly. In regex, [ ] means "pick one from this group," { } means "repeat this many times," and . means "any single character." Learn these instructions and you can write any recipe.

The Main Special Characters

Character Name Meaning Example
. Dot / Period Matches any single character (except newline) c.t matches "cat", "cot", "cut"
[ ] Square Brackets Matches one character from the given set [abc] matches "a", "b", or "c"
[^ ] Negated Brackets Matches any character NOT in the set [^abc] matches "d", "x", "1", etc.
( ) Parentheses Groups patterns together; captures matches (ab)+ matches "ab", "abab", "ababab"
{ } Curly Braces Quantifier — specifies how many times to repeat a{3} matches "aaa"
\ Backslash Escapes special characters; also starts shortcut classes \. matches a literal dot
| Pipe OR operator — matches either side cat|dog matches "cat" or "dog"
^ Caret Matches the start of the string ^Hello matches strings starting with "Hello"
$ Dollar Sign Matches the end of the string world$ matches strings ending with "world"
* Asterisk Zero or more repetitions of the preceding element ab* matches "a", "ab", "abb", "abbb"
+ Plus One or more repetitions of the preceding element ab+ matches "ab", "abb", "abbb" (not "a")
? Question Mark Zero or one repetition (makes it optional) colou?r matches "color" and "colour"
Using Special Characters Literally: If you want to match a special character itself (like a dot in an IP address or a plus in a math expression), you must escape it with a backslash. For example, to match the literal "+" character, write \+. To match a literal dot, write \..

The Basic Pattern Structure

Most regex patterns follow this general structure:

[ ]
Character Class
(what to match)
{ }
Quantifier
(how many times)

This [character class]{quantifier} pattern is the most common building block. The character class defines what characters are allowed, and the quantifier defines how many of them are expected.

Example: [a-z]{3} — matches exactly 3 lowercase letters
Matches: "cat", "dog", "the", "end"
No match: "ab" (too short), "ABCD" (uppercase), "123" (digits)

3. Character Classes

▼

Character classes are defined using square brackets [ ]. They tell the regex engine to match exactly one character from the specified set.

Analogy — Character Classes are Like Multiple Choice: In a multiple-choice question, you can pick one answer from the options A, B, C, or D. A character class works the same way — it says "this position can be any ONE of these characters." [abc] means "a" OR "b" OR "c" — just like picking one answer from a multiple-choice question.

Basic Character Classes

Pattern Meaning Matches Doesn't Match
[abc] Any one of a, b, or c "a", "b", "c" "d", "A", "1"
[a-z] Any lowercase letter "a" to "z" "A", "1", "@"
[A-Z] Any uppercase letter "A" to "Z" "a", "1", "@"
[0-9] Any digit "0" to "9" "a", "A", "@"
[a-zA-Z] Any letter (upper or lower) "a" to "z", "A" to "Z" "1", "@"
[a-zA-Z0-9] Any letter or digit "a-z", "A-Z", "0-9" "@", "!", " "

Negated Character Classes

Putting a ^ inside the square brackets negates the class — it matches any character not in the set.

Pattern Meaning Matches Doesn't Match
[^abc] Any character EXCEPT a, b, c "d", "1", "@", " " "a", "b", "c"
[^0-9] Any non-digit character "a", "A", "@", " " "0" to "9"
[^a-z] Any non-lowercase character "A", "1", "@", " " "a" to "z"
Tip — Ranges Can Be Combined: You can combine ranges inside a single pair of square brackets. [a-zA-Z0-9] matches any letter or digit. [aeiou] matches any vowel. The key is: the character class matches exactly one character from whatever you put inside the brackets.

Examples in Action

Pattern: [A-Z][a-z]+
Meaning: One uppercase letter followed by one or more lowercase letters
Matches: "Hello", "World", "Java"
No match: "hello" (starts lowercase), "JAVA" (all uppercase), "123" (digits)

4. Shortcut (Shorthand) Classes

▼

Regex provides shorthand notations for commonly used character classes. Instead of writing [0-9], you can simply write \d. These shortcuts make your patterns shorter and more readable.

Analogy — Shorthand is Like Abbreviations: Just like "Dr." is shorthand for "Doctor" and "NYC" is shorthand for "New York City," regex shorthand classes are abbreviations for common character classes. \d is short for [0-9], \w is short for [a-zA-Z0-9_]. They mean the same thing — one is just faster to write.
Shortcut Equivalent Meaning Example Matches
. (any character) Matches any single character except newline (\n) "a", "1", "@", " " (space)
\d [0-9] Matches any digit "0", "5", "9"
\D [^0-9] Matches any non-digit character "a", "A", "@", " "
\s [ \t\n\r\f] Matches any whitespace character (space, tab, newline, etc.) " " (space), "\t" (tab), "\n" (newline)
\S [^ \t\n\r\f] Matches any non-whitespace character "a", "1", "@", "!"
\w [a-zA-Z_0-9] Matches any word character (letters, digits, underscore) "a", "Z", "0", "_"
\W [^a-zA-Z_0-9] Matches any non-word character "@", "!", " ", "-"
Case Matters! In regex, uppercase and lowercase shortcuts have opposite meanings. \d matches digits, but \D matches non-digits. \w matches word characters, but \W matches non-word characters. Always pay attention to the case.

Practical Examples

Pattern: \d{3}-\d{4}
Meaning: Three digits, a hyphen, four digits (like a phone number)
Matches: "123-4567", "555-0199"
No match: "12-34567" (wrong digit count), "abc-defg" (not digits)
Pattern: \w+@\w+\.\w+
Meaning: word chars, @, word chars, dot, word chars (simple email-like)
Matches: "user@domain.com", "test@mail_org.net"
No match: "@domain.com" (missing username), "user@.com" (missing domain)

5. Repetition Quantifiers

▼

Quantifiers specify how many times a character, group, or character class must be present for a match. They are placed after the character or group they modify.

Analogy — Quantifiers are Like Quantity Orders: Imagine ordering food at a restaurant. You might say "I want 1 pizza" (exactly one), "I want at least 2 pizzas" (2 or more), or "I want 2 to 5 pizzas" (a range). Quantifiers work the same way for regex — they tell the engine exactly how many of the preceding element you need.
Quantifier Meaning Example Pattern Matches
p+ One or more (at least 1) a+ "a", "aa", "aaa", ... (not empty string)
p* Zero or more (can be absent) a* "" (empty), "a", "aa", "aaa", ...
p? Zero or one (optional) colou?r "color", "colour" (the "u" is optional)
p{n} Exactly n times a{3} "aaa" only (exactly 3 a's)
p{min,max} Between min and max times a{2,5} "aa", "aaa", "aaaa", "aaaaa"
p{min,} At least min times (no upper limit) a{2,} "aa", "aaa", "aaaa", ... (not "a")

Key Differences to Understand

+ vs *:
a+ — requires at least ONE "a": matches "a", "aa", "aaa"
a* — allows ZERO "a"s: matches "", "a", "aa", "aaa"

{n} vs {min,max}:
a{3} — exactly 3: matches only "aaa"
a{2,5} — 2 to 5: matches "aa", "aaa", "aaaa", "aaaaa"
Tip — {n} is a Common Pattern: The pattern p{3} means "exactly 3 of p." This is extremely useful. For example, \d{6} matches exactly 6 digits, [a-z]{3} matches exactly 3 lowercase letters, and \w{4,8} matches between 4 and 8 word characters.

6. Position Anchors

▼

Anchors don't match characters — they match positions in the string. They let you say "the pattern must be at the start" or "the pattern must be at the end."

Analogy — Anchors are Like Bookmarks: Imagine you're reading a book and you place a bookmark at the beginning of a chapter. The ^ anchor is like saying "this word must appear right at the bookmark (the start)." The $ anchor is like saying "this word must appear at the very last page (the end)." Anchors don't care about what the text is — they care about where the text is.
Anchor Position Example Matches
^ Start of the string (or line) ^Hello "Hello World" — yes | "Say Hello" — no
$ End of the string (or line) world$ "Hello world" — yes | "world Hello" — no

Using Both Anchors Together

When you use both ^ and $, the pattern must match the entire string from start to end. This is very useful for validation.

Pattern: ^[A-Z][a-z]+$
Meaning: String must start with an uppercase letter, followed by one or more lowercase letters, and end there.
Matches: "Hello", "World", "Alice"
No match: "hello" (starts lowercase), "Hello World" (has space), "HELLO" (uppercase at end)
Note on Multi-line Mode: In most regex engines, ^ and $ match the start/end of the entire string by default. In "multi-line" mode, they also match the start/end of each line within the string. For form validation, you usually use the default (entire string) mode.

7. Groups, Alternation, and Flags

▼

Capture Groups ( )

Parentheses ( ) create groups in your pattern. Groups serve two purposes: they let you apply quantifiers to an entire group, and they "capture" matched text for later use.

Pattern: (ha)+
The + applies to the entire group "ha", so it matches:
"ha", "haha", "hahaha", ...

Alternation (OR) with Pipe |

The pipe | acts as an OR operator inside groups or patterns.

Pattern: (cat|dog)
Matches: "cat" or "dog"

Pattern: I like (cats|dogs|birds)
Matches: "I like cats", "I like dogs", "I like birds"

Case-Insensitive Flag (?i)

By default, regex is case-sensitive. "A" and "a" are different. Adding (?i) at the start of your pattern makes it case-insensitive.

Pattern: (?i)hello
Matches: "hello", "Hello", "HELLO", "hElLo"
Tip — Combining with Other Features: You can combine groups with quantifiers and alternation. For example: ^(Mon|Tue|Wed|Thu|Fri) matches strings starting with a weekday abbreviation. \d{2,4}(-\d{2,4})? matches "12-34", "1234-5678", or just "1234".

8. Practical Regex Patterns

▼

Let's put everything together with real-world patterns you'll actually use.

Matric Number Validation

Pattern: [aA][1][0-9]{6}
[aA]
Starts with "a" or "A"
[1]
Followed by literal "1"
[0-9]{6}
Exactly 6 digits
Matches: "a1234567", "A1000001"
No match: "b1234567" (wrong letter), "a123456" (only 5 digits), "a12345678" (8 digits)

Date Validation (DD.MM.YYYY)

Pattern: (0[1-9]|1[0-9]|2[0-9]|3[01])\.(0[1-9]|1[012])\.[0-9]{4}
(0[1-9]|1[0-9]|2[0-9]|3[01])
Day: 01–31
\.
Literal dot
(0[1-9]|1[012])
Month: 01–12
\.
Literal dot
[0-9]{4}
Year: 4 digits
Matches: "25.12.2024", "01.01.2000"
No match: "32.01.2024" (day 32), "25.13.2024" (month 13), "5.1.2024" (missing leading zeros)

Malaysian Phone Number

Pattern: \+60[0-9]{2}\-[0-9]{7}
\+
Literal "+" sign
60
Malaysia code
[0-9]{2}
2 digits (area/provider)
\-
Literal hyphen
[0-9]{7}
7 digits (number)
Matches: "+6012-3456789", "+6019-8765432"
No match: "012-3456789" (missing +60), "+60123456789" (missing hyphen)

Spam/Scam Detection (Viagra)

Why this matters: Spammers use character substitutions (l33t speak) to bypass simple filters. Regex helps catch these tricks.
Pattern: v[i!1][a@]gr[a@]
v
Literal "v"
[i!1]
"i" or "!" or "1"
[a@]
"a" or "@"
gr
Literal "gr"
[a@]
"a" or "@"
Matches: "viagra", "v!agra", "v1@gr@", "vi@gra"
Catches all common spammer substitutions!

String Ending with .com

Pattern: .*(\.com)$
Meaning: Any characters, followed by a literal ".com" at the end of the string
Matches: "google.com", "www.example.com", "site-name.com"
No match: "example.com.org", "example.com/bad"

String Starting with "The"

Pattern: ^The
Meaning: The string must begin with "The"
Matches: "The cat sat", "There once was..."
No match: "A the cat", "them", "other"

10. Flashcards — Key Concepts

▼
What does . (dot) match in regex?
Any single character except the newline character (\n).
What is the difference between + and *?
+ = one or more (at least 1).
* = zero or more (can be 0).
What does \d match?
Any digit — equivalent to [0-9].
What do ^ and $ do?
^ matches the start of the string.
$ matches the end of the string.
How do you match a literal "+" in regex?
Escape it: \+. The backslash removes the special meaning.
What does [^abc] match?
Any character that is NOT "a", "b", or "c". The ^ inside brackets means "NOT".
What does {2,5} mean?
Match the preceding element 2 to 5 times (inclusive).
What does (cat|dog) match?
Either "cat" or "dog". The pipe | is the OR operator.

Summary — Regular Expressions

  • Regex is a powerful pattern-matching tool used for text verification, searching, and replacement.
  • Character Classes [ ] match one character from a set. Use ranges like [a-z], [0-9], and negation [^ ].
  • Shortcut Classes: \d (digit), \w (word char), \s (whitespace), . (any char). Uppercase versions negate.
  • Quantifiers: + (1+), * (0+), ? (0 or 1), {n} (exactly n), {min,max} (range).
  • Anchors: ^ (start), $ (end) — match positions, not characters.
  • Groups & OR: ( ) groups patterns, | provides alternation (OR).
  • Escaping: Use \ before special characters to match them literally.
  • Practical patterns: Matric numbers, dates, phone numbers, spam detection, email-like patterns.
