Topic 6: PHP Fundamentals

The Restaurant Analogy — PHP as a Chef: Think of building a website like running a restaurant. HTML is the menu — it shows what's available. CSS is the plating — it makes the food look beautiful. But PHP? PHP is the chef in the kitchen. When a customer (the user) sends an order (a request), the chef (PHP) reads the order, gathers ingredients from the pantry (variables, database), prepares the dish (processes logic), and sends it back to the customer (response). The customer never sees the kitchen — they just get their freshly prepared meal. That's how PHP works: it runs on the server, processes everything behind the scenes, and sends back the final HTML to the user's browser.

1. What is PHP?

▼

The Name

PHP originally stood for "Personal Home Page". It was created by Rasmus Lerdorf in 1994. The name was later officially changed to "PHP: Hypertext Preprocessor" — a recursive acronym (PHP stands for PHP Hypertext Preprocessor, which stands for PHP Hypertext Preprocessor... and so on!).

Analogy — The Name: "Hypertext Preprocessor" literally tells you what PHP does: it preprocesses hypertext. Just like a chef prepares food before it reaches the customer, PHP processes code before the HTML reaches the browser.

How to Get PHP

You don't download PHP in isolation. You install an all-in-one package that includes PHP, a web server (Apache), and a database (MySQL):

  • XAMPP — Most popular, works on Windows/Mac/Linux. Recommended for beginners.
  • WAMP — Windows only (Apache + MySQL + PHP).
  • LAMP — Linux only (Apache + MySQL + PHP).
Tip: After installing XAMPP, start the Apache service. Place your .php files in the C:\xampp\htdocs\ folder and access them via http://localhost/yourfile.php in your browser.

PHP File Processing

PHP files go through the Zend Engine, which processes them as either:

  • Module — PHP runs as a module inside the web server (Apache). This is the most common and fastest method.
  • Daemon/Service — PHP runs as a background service that the web server communicates with.
  • CGI — PHP runs as a separate program for each request. Slower, but more isolated.

The chef reads the order, cooks the dish (executes PHP code), plates it (generates HTML), and the waiter brings it back (server sends response). The customer gets the finished dish and never sees the kitchen.

3. PHP Syntax Basics

▼

PHP Tags

PHP code is enclosed within special tags:

<?php // Your PHP code goes here ?>

The <?php tag tells the server "start processing PHP here." The ?> tag says "go back to HTML mode."

Important: In well-formed PHP files that contain only PHP code (no HTML), the closing ?> tag is often omitted. This is considered a best practice as it prevents accidental whitespace output. For files mixing PHP and HTML, use both tags.

Statements

Every PHP statement ends with a semicolon (;). This is similar to C and Java.

<?php echo "Hello World"; // This is a statement $x = 5; // This is also a statement echo "x = " . $x; // Another statement ?>

Comments

Comments are ignored by the PHP engine. They're for human readers:

<?php // This is a single-line comment # This is also a single-line comment /* This is a multi-line comment */ ?>
Analogy — Syntax Like a Recipe: PHP syntax is similar to C. The semicolon is like a period at the end of a sentence — it tells PHP "this instruction is done." Comments are like sticky notes on a recipe: "don't forget to preheat the oven." The chef (PHP) ignores the sticky notes and just follows the instructions.

4. Variables

▼

Variables are named containers that store data. In PHP, every variable starts with a $ sign.

Analogy — Variables as Containers: Imagine the chef's kitchen has labeled containers on the shelf. Each container has a name tag and holds something: a container labeled "flour" holds flour, one labeled "sugar" holds sugar. In PHP, $name is like a container that can hold a value. You put a value in with =: $name = "Alice"; puts "Alice" into the container called "name."

Variable Rules

Loosely Typed — No Declaration Needed

PHP is loosely typed, meaning you don't need to declare a variable's type. You just assign a value and PHP figures out the type automatically.

<?php $name = "Alice"; // String $age = 25; // Integer $price = 19.99; // Float $isActive = true; // Boolean // PHP determined the type from the value assigned! // No need to write: int $age = 25; (like in C or Java) ?>
Note: The $ is part of the variable name in PHP, not a type indicator. In some other languages (like JavaScript), $ is just a prefix convention, but in PHP it's required syntax.

5. Output — echo and print

▼

To display output in PHP, you use echo or print. These send text back to the browser.

Analogy — echo as Plating: After the chef finishes cooking (processing PHP), they need to plate the dish and send it out. echo is like plating food — it takes what you've prepared and sends it out to the customer (the browser). echo is fast and can serve multiple items at once.

echo

echo is the most commonly used output statement. It can output multiple values separated by commas (or dots for concatenation).

<?php // Simple output echo "Hello World!"; // Output with variable $name = "Alice"; echo "Hello, " . $name . "!"; // Multiple outputs echo "First", "Second", "Third"; // HTML inside echo echo "<h1>This is a heading</h1>"; echo "<p style='color:red;'>Red text</p>"; ?>

print

print works similarly to echo but outputs only one value and returns 1 (true).

<?php print "Hello World!"; print "This also works"; // print cannot do: print "a", "b"; — this causes an error ?>

String Concatenation (The Dot Operator)

In PHP, the . (dot) is used to join strings together. This is different from many other languages that use +.

<?php $first = "Hello"; $second = "World"; echo $first . " " . $second; // Output: Hello World // Mixing strings and numbers $age = 25; echo "I am " . $age . " years old."; ?>
// echo vs print echo "Speed: "; // Faster print "Speed: "; // Slightly slower echo "A", "B"; // ✓ OK print "A", "B"; // ✗ Error echo strlen("Hi"); // Returns 2 print strlen("Hi"); // Returns 1 (print returns 1)
Tip: In practice, echo is preferred over print because it's slightly faster and can output multiple values. Most PHP developers use echo exclusively.

6. Variable Scope

▼

Variable scope determines where a variable can be accessed. PHP has three types of scope:

Analogy — Kitchen Zones: Think of the kitchen as having zones. Global scope is like the main kitchen counter — everything on it is accessible to everyone. Local scope is like a personal cutting board inside a specific station — only the chef at that station can see it. Static is like a special container that remembers its contents even after the chef leaves the station.

Global Scope

Variables declared outside a function. Accessible everywhere except inside functions (unless you use the GLOBAL keyword).

Local Scope

Variables declared inside a function. Accessible only within that function.

Static Scope

Variables declared with the static keyword inside a function. They retain their value between function calls.

Global Variables

<?php $x = 10; // Global scope function myTest() { // $x is NOT accessible here (local scope) echo "x = " . $x; // ERROR: Undefined variable } myTest(); ?>

Accessing Global Variables Inside Functions

<?php $x = 10; function myTest() { global $x; // Bring global variable into function scope echo "x = " . $x; // Output: x = 10 } myTest(); ?>

Local Variables

<?php function greet() { $message = "Hello!"; // Local scope echo $message; // Works here } greet(); echo $message; // ERROR: $message is not defined here ?>

Static Variables

<?php function counter() { static $count = 0; // Retains value between calls $count++; echo "Count: " . $count . "<br>"; } counter(); // Count: 1 counter(); // Count: 2 counter(); // Count: 3 // Without static, each call would reset to 1 ?>
Common Pitfall: Forgetting to use the global keyword when accessing a global variable inside a function is one of the most common PHP mistakes. Always check your scope!

7. Data Types

▼

PHP supports several data types. Since PHP is loosely typed, you don't declare types — PHP determines them automatically.

Analogy — Data Types as Ingredients: Just like a chef handles different ingredients differently — flour (powder) is scooped, eggs (liquid) are cracked, vegetables (solid) are chopped — PHP handles different data types differently. You don't tell PHP "this is an integer." You just pour an integer in, and PHP knows how to handle it.
Data Type Description Example
String Sequence of characters (text) $name = "Alice";
Integer Whole number (no decimals) $age = 25;
Float Number with decimals $price = 19.99;
Boolean True or false $active = true;
Array Collection of values $colors = ["red", "blue"];
Object Instance of a class $car = new Car();
NULL Empty / no value $x = null;

Checking Types with var_dump()

var_dump() shows you the value and type of a variable — very useful for debugging.

<?php $name = "Alice"; $age = 25; $price = 19.99; $active = true; $nothing = null; var_dump($name); // string(5) "Alice" var_dump($age); // int(25) var_dump($price); // float(19.99) var_dump($active); // bool(true) var_dump($nothing); // NULL ?>

Type Casting and Changing Types

<?php // Type casting (like in C) $x = (int) "123"; // $x is now integer 123 $y = (string) 456; // $y is now string "456" // Using settype() $z = "789"; settype($z, "integer"); // $z is now integer 789 var_dump($x); // int(123) var_dump($y); // string(3) "456" var_dump($z); // int(789) ?>

Analogy — Built-in Functions are Like Kitchen Tools: A chef doesn't chop every vegetable by hand — they have a knife, a blender, a grater. Each tool does a specific job quickly. PHP's built-in functions are your kitchen tools: strlen() is a ruler (measures length), strrev() is a mirror (reverses text), strpos() is a magnifying glass (finds things).

9. Arrays

▼

Arrays store multiple values in a single variable. Instead of creating $color1, $color2, $color3, you can store all colors in one $colors array.

Analogy — Arrays as Spice Racks: A spice rack holds multiple spices in organized slots. Each slot has a number (or name). You can add new spices, remove old ones, and look up any spice by its slot. PHP arrays work the same way — they hold multiple values, each accessible by a key (index or name).

Three Types of Arrays

1. Numeric Arrays

Each element has a numeric index (starting from 0).

<?php // Method 1: Using array() function $colors = array("Red", "Green", "Blue"); // Method 2: Using square brackets (PHP 5.4+) $fruits = ["Apple", "Banana", "Cherry"]; // Access elements (0-based index) echo $colors[0]; // Red echo $colors[1]; // Green echo $colors[2]; // Blue ?>

2. Associative Arrays

Each element has a named key (string index) instead of a number.

<?php $person = array( "name" => "Alice", "age" => 25, "city" => "KL" ); // Access by key name echo $person["name"]; // Alice echo $person["age"]; // 25 ?>

3. Multidimensional Arrays

Arrays can contain other arrays — like a filing cabinet with folders inside folders.

<?php $students = array( array("name" => "Alice", "grade" => "A"), array("name" => "Bob", "grade" => "B+"), array("name" => "Charlie", "grade" => "A-") ); echo $students[0]["name"]; // Alice echo $students[1]["grade"]; // B+ ?>

The foreach Loop

foreach is the easiest way to loop through all elements in an array.

<?php $colors = ["Red", "Green", "Blue"]; foreach ($colors as $color) { echo $color . "<br>"; } // Output: Red, Green, Blue (each on a new line) // With associative arrays - use as $key => $value $person = ["name" => "Alice", "age" => 25]; foreach ($person as $key => $value) { echo $key . ": " . $value . "<br>"; } // Output: name: Alice, age: 25 ?>

11. Conditional Statements (if / elseif / else)

▼

Conditional statements let your PHP code make decisions. The syntax is similar to C and Java.

Analogy — Conditionals as Recipe Steps: A recipe might say: "If the oven is preheated, put the cake in. Otherwise, wait. If you're making a chocolate cake, use cocoa powder; if vanilla, use vanilla extract." PHP conditionals work exactly the same way — they check conditions and execute different code paths.

Basic if-else Structure

<?php $temperature = 32; if ($temperature > 30) { echo "It's hot outside!"; } elseif ($temperature > 20) { echo "It's nice outside."; } else { echo "It's cold outside."; } // Output: It's hot outside! ?>

Mixing PHP and HTML

<?php $hour = date("H"); // Current hour (0-23) ?> <!-- Now back to HTML --> <h1>Welcome!</h1> <?php if ($hour < 12) { echo "<p>Good Morning!</p>"; } elseif ($hour < 18) { echo "<p>Good Afternoon!</p>"; } else { echo "<p>Good Evening!</p>"; } ?> <!-- More HTML below --> <p>This is always visible.</p>
Note: You can switch between PHP and HTML as many times as you want in a single file. PHP starts at <?php and ends at ?>, while everything else is treated as plain HTML.

12. User-Defined Functions

▼

Functions are blocks of reusable code. Instead of writing the same code multiple times, you put it in a function and call it whenever needed.

Analogy — Functions as Recipes: A function is like a recipe card. The chef writes down the recipe once (function definition), then whenever a customer orders that dish, the chef just follows the recipe card (function call). If the customer wants extra spice, they pass it as a request (parameters). The finished dish is what comes back (return value).

Basic Function

<?php // Define the function function greet() { echo "Hello World!"; } // Call the function greet(); // Output: Hello World! greet(); // Output: Hello World! (can call multiple times) ?>

Function with Parameters

<?php function greetUser($name) { echo "Hello, " . $name . "!"; } greetUser("Alice"); // Hello, Alice! greetUser("Bob"); // Hello, Bob! ?>

Function with Return Value

<?php function add($a, $b) { return $a + $b; } $result = add(5, 3); echo "5 + 3 = " . $result; // 5 + 3 = 8 // You can use the function directly in echo echo "10 + 7 = " . add(10, 7); // 10 + 7 = 17 ?>

Practical Example — Reusable Utility

<?php function isEven($number) { return $number % 2 === 0; } if (isEven(4)) { echo "4 is even"; } if (isEven(7)) { echo "7 is even"; } else { echo "7 is odd"; // This prints } ?>

date() formats a timestamp into a readable date/time string. It's one of PHP's most commonly used functions.

strtotime() converts a text string into a Unix timestamp (number of seconds since Jan 1, 1970).

14. Including Files (include / require)

▼

PHP lets you insert the content of one file into another using include or require. This is essential for organizing code — you can write common code once and reuse it everywhere.

Analogy — Include as Prep Work: A restaurant doesn't make everything from scratch for every order. They prep common ingredients in advance (chopped onions, sauces, bread). When an order comes, the chef grabs the prepped items (include) and assembles the dish. include is like reaching into the prep station and grabbing pre-made components to use in your current dish.

The Two Commands

Command On Failure Use When...
include Shows a warning, script continues The file is optional (e.g., footer, sidebar)
require Shows a fatal error, script stops The file is essential (e.g., database config)

Example

Imagine you have a file called header.php:

<!-- header.php --> <html> <head> <title>My Website</title> </head> <body> <nav>...navigation menu...</nav>

And a file called footer.php:

<!-- footer.php --> <footer> <p>Copyright 2024 My Website</p> </footer> </body> </html>

Now your main page index.php can include both:

<!-- index.php --> <?php include 'header.php'; ?> <h1>Welcome to My Page!</h1> <p>This is the main content.</p> <?php include 'footer.php'; ?>
Tip: Use require_once or include_once to ensure a file is included only once, even if you accidentally call it multiple times. This prevents "Cannot redeclare function" errors.

15. GET vs POST — Form Data Handling

▼

When a user submits an HTML form, the data is sent to the server via either GET or POST methods. PHP provides superglobals $_GET and $_POST to access this data.

Analogy — GET vs POST as Ways to Place an Order: GET is like writing your order on a visible whiteboard that everyone in the restaurant can see — the order, your table number, everything is out in the open. POST is like putting your order in a sealed envelope — it's private, no one else sees it, and you can attach large documents (files).
Feature GET POST
Data visibility Visible in URL (query string) Not visible in URL (sent in body)
URL example page.php?name=Alice&age=25 page.php (clean URL)
Size limit ~2000 characters max No practical limit
Bookmarkable Yes (URL contains data) No
File upload Not supported Supported
Security Less secure (data in URL) More secure (data hidden)
Best for Search, filtering, pagination Login forms, file uploads, sensitive data

GET Example

HTML form:

<!-- search.html --> <form action="search.php" method="GET"> <input type="text" name="query"> <button type="submit">Search</button> </form>

PHP handling:

<!-- search.php --> <?php $query = $_GET["query"]; echo "You searched for: " . $query; ?>

If the user types "hello world", the URL becomes: search.php?query=hello+world

POST Example

HTML form:

<!-- login.html --> <form action="login.php" method="POST"> <input type="text" name="username"> <input type="password" name="password"> <button type="submit">Login</button> </form>

PHP handling:

<!-- login.php --> <?php $user = $_POST["username"]; $pass = $_POST["password"]; echo "Welcome, " . $user; // Password is NOT visible in the URL! ?>
Security Warning: Never trust user input! Always validate and sanitize data from both $_GET and $_POST before using it. Use functions like htmlspecialchars() to prevent XSS attacks, and use prepared statements when working with databases.

Summary — PHP Fundamentals

  • PHP (Hypertext Preprocessor) is a server-side language that processes code on the server and sends HTML output to the browser.
  • PHP Tags: Code goes between <?php and ?>. Statements end with ;.
  • Variables start with $ and are loosely typed (no type declaration needed).
  • Output: echo (most common, multiple values) and print (single value, returns 1).
  • Scope: Global (outside functions), Local (inside functions), Static (retains value). Use global keyword to access globals inside functions.
  • Data Types: String, Integer, Float, Boolean, Array, Object, NULL. Use var_dump() to check.
  • String Functions: strlen(), str_word_count(), strrev(), strpos(), str_replace().
  • Arrays: Numeric (indexed), Associative (named keys), Multidimensional. Use foreach to loop.
  • Operators: Arithmetic, Comparison (=== for strict), String (. for concatenation), Logical, Ternary.
  • Functions: Use function keyword. Support parameters and return values.
  • Date Functions: date() for formatting, strtotime() for parsing text to timestamps.
  • Include/Require: Insert file contents. require = fatal on failure, include = warning only.
  • GET vs POST: GET visible in URL (~2000 char limit), POST hidden and supports file upload.
