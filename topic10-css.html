CSS — Cascading Style Sheets

CSS describes how HTML elements look on screen. It controls layout, colours, fonts, spacing, and responsive design across your entire website.

Topic 10 of 14 — 71%

Learning Outcomes

After this topic you will be able to:

  • Explain what CSS is and why it was created
  • Write CSS rules using selectors, properties, and values
  • Apply CSS using external, internal, and inline methods
  • Calculate CSS specificity to resolve conflicting rules
  • Understand the Box Model: content, padding, border, margin
  • Style text, fonts, links, backgrounds, borders, and tables
  • Use CSS display and position properties to control layout
  • Apply colours and measurements in CSS

1. What is CSS?

CSS = Cascading Style Sheets

CSS describes how HTML elements should be rendered on screen, paper, or other media. It lets you control the layout and appearance of multiple pages from a single file.

External stylesheets are stored in .css files and linked to HTML documents.

Analogy — Clothes on a Mannequin: HTML is the mannequin (the structure — arms, legs, head). CSS is the clothing (the style — colour, size, position). Without CSS, every mannequin looks the same (plain, unstyled). With CSS, you can dress each mannequin differently — or style them all consistently with one wardrobe.

Why Was CSS Created?

HTML was never meant for formatting. In the early days, developers used tags like <font>, <center>, and <b> directly in HTML. This caused nightmares:

  • Every page had to be styled individually (massive repetition)
  • Changing a site's look meant editing every single HTML file
  • HTML became cluttered with presentation code mixed with content

The W3C (World Wide Web Consortium) created CSS to separate style from structure, solving all of these problems.

Tip: Modern best practice: use HTML for structure/meaning and CSS for all visual presentation. Never use <font> or inline style attributes for real projects.

2. CSS Syntax

Anatomy of a CSS Rule

selector {
    property: value;
    property: value;
}

A CSS rule consists of:

  • Selector — points to the HTML element(s) to style
  • Declaration Block — curly braces { } containing declarations
  • Declaration — a property: value; pair
h1 {
    color: blue;
    font-size: 24px;
    text-align: center;
}

Here h1 is the selector, and there are three declarations. Every declaration ends with a semicolon.

Analogy — Recipe: The selector is the ingredient you're cooking (e.g., "pasta"). The property is what aspect you're changing (e.g., "cook time"). The value is the setting (e.g., "8 minutes"). One ingredient (selector) can have multiple cooking instructions (declarations).

3. CSS Selectors

Types of Selectors

SelectorSyntaxWhat It SelectsExample
Element element All instances of an HTML element p { color: red; } — all paragraphs
ID #id A single unique element with that ID #sidebar { width: 250px; }
Class .class All elements with that class name .wrapper { max-width: 1200px; }
Descendant A B All B elements inside A (any depth) div p { margin: 10px; }
Grouped A, B Elements matching A OR B h1, p.wrapper { font-family: Arial; }
Attribute [attr] Elements with that attribute a[target="_blank"] { color: red; }

More Selector Examples

/* Element selector */
p {
    line-height: 1.6;
}

/* ID selector - use for unique elements */
#header {
    background-color: #333;
    color: white;
}

/* Class selector - use for reusable styles */
.highlight {
    background-color: yellow;
    font-weight: bold;
}

/* Descendant selector - p inside div */
div p {
    padding-left: 20px;
}

/* Grouped selector - apply same style to multiple */
h1, h2, h3 {
    font-family: 'Georgia', serif;
}

/* Attribute selector */
input[type="text"] {
    border: 1px solid #ccc;
}

a[target="_blank"]::after {
    content: " (opens in new tab)";
}
Important: IDs must be unique on a page (use once). Classes can be reused (use many times). Choose class over ID for styling when possible — classes are more flexible and have lower specificity.

4. Three Ways to Insert CSS

Method 1: External Stylesheet (Recommended)

A separate .css file linked in the <head>. This is the best practice for all real projects.

<head>
    <link rel="stylesheet" href="style.css">
</head>

Pros: Single file styles all pages, easy to maintain, cached by browser for faster loading.

Method 2: Internal Stylesheet

CSS written inside a <style> tag in the <head>. Useful for page-specific styles.

<head>
    <style>
        body {
            background-color: #f0f0f0;
            font-family: Arial, sans-serif;
        }
    </style>
</head>

Pros: No extra file needed. Cons: Only affects one page, can't be cached.

Method 3: Inline Styles

CSS applied directly on an element using the style attribute. Avoid in real projects.

<p style="color: red; font-size: 18px;">This text is red and 18px.</p>

Pros: Quick for testing. Cons: Mixes style with structure, hard to maintain, overrides everything (too specific).

Analogy — Dress Code: External = a company-wide dress code policy (everyone follows one document). Internal = a department-specific addendum (only that department). Inline = a sticky note on your desk saying "wear a hat today" (only affects you, overrides everything).

5. CSS Specificity

What is Specificity?

When multiple CSS rules target the same element, specificity determines which rule wins. Think of it as a ranking system.

Specificity Hierarchy (highest to lowest)

RankSelector TypeExampleSpecificity Value
1 (Highest)Inline stylestyle="..."1,0,0,0
2ID selector#header0,1,0,0
3Class / Attribute / Pseudo-class.nav, [type], :hover0,0,1,0
4 (Lowest)Element / Pseudo-elementp, ::before0,0,0,1

Rules for Resolving Conflicts

  1. A more specific selector always wins (ID beats class beats element)
  2. If specificity is equal, the last rule wins (cascade order)
  3. !important overrides everything (avoid using it — it's a maintenance nightmare)
/* This targets ALL paragraphs */
p {
    color: black;        /* specificity: 0,0,0,1 */
}

/* This targets paragraphs with class "intro" */
p.intro {
    color: blue;         /* specificity: 0,0,1,1 */
}

/* This targets the element with ID "main" */
#main p {
    color: green;        /* specificity: 0,1,0,1 */
}

/* Result: paragraphs inside #main are green (highest specificity) */
Analogy — Chain of Command: Specificity is like a chain of command in the military. A general's order (ID selector) always overrides a sergeant's order (class selector), which overrides a private's request (element selector). If two generals give conflicting orders, the one who speaks last wins.

6. The Box Model

Every Element is a Box

In CSS, every HTML element is treated as a rectangular box with four layers, from inside to outside:

Margin — transparent space outside the border
Border — the visible edge around the element
Padding — space between content and border
Content — the actual text, image, or media

Key Facts

  • width and height only set the content area size
  • Actual visible width = width + padding-left + padding-right + border-left + border-right
  • Total space used = content + padding + border + margin
.box {
    width: 300px;
    padding: 20px;
    border: 5px solid black;
    margin: 10px;
}

/* Actual rendered width = 300 + 20 + 20 + 5 + 5 = 350px */
/* Total space occupied  = 350 + 10 + 10 = 370px */

Box-Sizing: The Fix

/* Use border-box so width INCLUDES padding and border */
* {
    box-sizing: border-box;
}

/* Now width: 300px means the total box is 300px,
   including padding and border */
Analogy — Gift Wrapped Package: The content is the actual gift. The padding is the bubble wrap around it. The border is the wrapping paper. The margin is the empty space between this gift and other gifts on the table. box-sizing: border-box is like measuring the whole package, wrapping paper and all.

7. CSS Colours

Three Ways to Specify Colours

MethodSyntaxExample
Named Colours colour name color: red;
HEX (Hexadecimal) #RRGGBB color: #ff0000;
RGBA rgba(R, G, B, A) color: rgba(255, 0, 0, 0.5);

Details

  • Named: 140 standard colour names (red, tomato, cornflowerblue, etc.)
  • HEX: 6-digit hexadecimal where each pair represents Red, Green, Blue (00–FF). #ff0000 = full red, #00ff00 = full green, #0000ff = full blue. #000000 = black, #ffffff = white.
  • RGBA: Same as RGB but with an alpha (opacity) value from 0 (transparent) to 1 (opaque). rgba(0, 0, 0, 0.5) = semi-transparent black.
h1 { color: tomato; }            /* Named */
p  { color: #333333; }           /* HEX */
a  { color: rgba(0, 0, 0, 0.7); }/* RGBA with 70% opacity */
Note: HEX is the most commonly used format in professional web development. Short-hand hex is also valid: #f00 is the same as #ff0000.

8. CSS Measurements

Relative vs Absolute Units

TypeUnitDescriptionUse Case
Relative px Pixels — fixed screen units Borders, precise sizing
em Relative to parent element's font size Spacing, padding, font sizes
rem Relative to root (<html>) font size Consistent font sizing
% Relative to parent element Widths, responsive layouts
Absolute in Inches Print stylesheets
cm Centimetres Print stylesheets
mm Millimetres Print stylesheets
pt Points (1pt = 1/72 inch) Print, typography
Tip: For web screens, use px for borders and precise values, em/rem for font sizes and spacing, and % for widths. Avoid cm, mm, in, and pt for screen layout — they're meant for print.

9. Backgrounds & Borders

Backgrounds

.hero {
    background-color: #3498db;              /* Solid colour */
    background-image: url('image.jpg');      /* Background image */
    background-size: cover;                  /* Fill the element */
    background-repeat: no-repeat;            /* Don't tile the image */
    background-position: center;             /* Centre the image */
}

/* Shorthand */
.hero {
    background: #3498db url('image.jpg') no-repeat center / cover;
}

Borders

A border requires at minimum a style (the other properties have defaults).

.card {
    border-width: 2px;
    border-style: solid;      /* Required! Options: solid, dashed, dotted, double, none */
    border-color: #333;
}

/* Shorthand */
.card {
    border: 2px solid #333;
}

/* Individual sides */
.card {
    border-top: 1px dashed red;
    border-bottom: 3px double blue;
    border-radius: 8px;    /* Rounded corners */
}

10. Margins & Padding

Margins

Space outside the border. Margins are always transparent.

.box {
    margin-top: 20px;
    margin-right: 10px;
    margin-bottom: 20px;
    margin-left: 10px;

    /* Shorthand (clockwise: top, right, bottom, left) */
    margin: 20px 10px 20px 10px;

    /* Two values: vertical | horizontal */
    margin: 20px 10px;

    /* One value: all sides */
    margin: 15px;

    /* Centre horizontally */
    margin: 0 auto;
}

Margins accept: auto (centres block elements), length (px, em, etc.), %, and inherit.

Padding

Space between content and border. Padding is affected by the element's background colour.

.card {
    padding-top: 10px;
    padding-right: 20px;
    padding-bottom: 10px;
    padding-left: 20px;

    /* Shorthand (same pattern as margin) */
    padding: 10px 20px;
}
Analogy — Picture Frame: The content is the photograph. The padding is the mat board between the photo and the frame. The border is the wooden frame itself. The margin is the wall space between this frame and the next picture hanging beside it.

11. Text Styling

Text Properties

p {
    color: #333;                          /* Text colour */
    text-align: center;                   /* left, right, center, justify */
    text-decoration: underline;           /* none, underline, overline, line-through */
    text-transform: uppercase;            /* capitalize, uppercase, lowercase */
    text-indent: 50px;                    /* First line indent */
    letter-spacing: 2px;                  /* Space between characters */
    line-height: 1.6;                     /* Space between lines (unitless = relative) */
    word-spacing: 5px;                    /* Space between words */
    direction: ltr;                       /* ltr or rtl (right-to-left languages) */
}

Font Properties

body {
    font-style: italic;       /* normal, italic, oblique */
    font-variant: small-caps; /* normal, small-caps */
    font-weight: bold;        /* normal, bold, 100-900 */
    font-size: 16px;          /* Size of the text */
    line-height: 1.5;         /* Spacing between lines */
    font-family: Arial, Helvetica, sans-serif;
}

/* Shorthand (order matters!) */
body {
    font: italic small-caps bold 16px/1.5 Arial, Helvetica, sans-serif;
}

Generic Font Families

FamilyDescriptionExample
serifSmall decorative strokes at the end of charactersTimes New Roman, Georgia
sans-serifClean, no decorative strokesArial, Helvetica, Verdana
cursiveHandwriting-styleComic Sans MS, Brush Script

Tip: Always end your font-family list with a generic family (e.g., sans-serif). If the user doesn't have your specific fonts installed, the browser falls back to the generic family.

12. Styling Links

The Four Link States (Order Matters!)

/* Unvisited link */
a:link {
    color: #3498db;
}

/* Visited link */
a:visited {
    color: #8e44ad;
}

/* Mouse hover */
a:hover {
    color: #e74c3c;
    text-decoration: none;
}

/* Active link (being clicked) */
a:active {
    color: #f39c12;
    transform: scale(1.05);
}

The L V HA Rule

To remember the order, think: LoVe HAcks (Link, Visited, Hover, Active). The hover rule must come after link and visited, and active must come after hover.

StateDescriptionPseudo-class
LinkNormal, unvisited linka:link
VisitedUser has visited this URLa:visited
HoverMouse is over the linka:hover
ActiveLink is being clicked (mouse down)a:active

13. Styling Tables

table {
    width: 100%;
    border-collapse: collapse;    /* Merges adjacent borders */
    font-family: Arial, sans-serif;
}

th, td {
    border: 1px solid #ddd;
    padding: 12px 15px;
    text-align: left;
}

th {
    background-color: #3498db;
    color: white;
    font-weight: bold;
}

tr:nth-child(even) {
    background-color: #f2f2f2;   /* Zebra striping */
}

tr:hover {
    background-color: #e0e0e0;
}

td {
    vertical-align: middle;      /* top, middle, bottom */
}

Outline vs Border

An outline is drawn outside the element's border. Unlike border, outline does not affect element dimensions — it doesn't take up space and doesn't affect layout.

input:focus {
    outline: 3px solid #3498db;
    outline-offset: 2px;    /* Space between border and outline */
}

input {
    outline: none;           /* Remove outline (use with caution for accessibility) */
}

14. CSS Display

Three Main Display Values

ValueBehaviourCommon Elements
block Takes full width, starts on a new line div, h1-h6, p, form, header, footer, section
inline Takes as much width as needed, stays in the same line span, a, img, strong, em
none Element is hidden and removed from document flow Any element (used for hiding/showing with JS) 
.hidden {
    display: none;           /* Completely removed from layout */
}

.invisible {
    visibility: hidden;      /* Hidden but still takes up space */
}

/* inline elements can't have width/height */
span {
    display: inline-block;   /* Inline behaviour but accepts width/height */
}

nav a {
    display: inline-block;
    padding: 10px 20px;
}
Analogy — Concert Seating: Block = a sofa that takes up the entire row and pushes everyone else to the next line. Inline = a chair that sits next to other chairs in the same row. Inline-block = a chair that sits in the row but also has adjustable width. None = an invisible seat — nobody can see it and no space is reserved.

15. CSS Position

Position Values

ValueBehaviourUse Case
static Default. Element follows normal document flow. Top/right/bottom/left have no effect. Normal page elements
relative Element stays in normal flow but is nudged from its original position using top/right/bottom/left. Positioning child elements, small adjustments
fixed Removed from document flow. Positioned relative to the viewport. Does not scroll with the page. Sticky headers, floating buttons, pop-ups
absolute Removed from document flow. Positioned relative to the nearest positioned ancestor (one with position set to relative, absolute, or fixed). If none exists, uses the viewport. Overlays, dropdowns, complex layouts
sticky Behaves like relative until a scroll threshold is met, then behaves like fixed. Sticky table headers, navigation that pins on scroll 
/* Static - default, no special behaviour */
.static-box {
    position: static;
}

/* Relative - shifted from its original position */
.relative-box {
    position: relative;
    top: 20px;       /* Moved 20px DOWN from where it would be */
    left: 10px;      /* Moved 10px RIGHT from where it would be */
}

/* Fixed - stays in place when scrolling */
.nav-bar {
    position: fixed;
    top: 0;
    left: 0;
    width: 100%;
    z-index: 1000;
}

/* Absolute - positioned relative to nearest positioned parent */
.tooltip {
    position: absolute;
    top: -30px;
    left: 50%;
    transform: translateX(-50%);
}

/* Sticky - relative until scroll threshold, then fixed */
.sticky-header {
    position: sticky;
    top: 0;
    background: white;
    z-index: 100;
}
Analogy — People in a Room: Static = standing in line like everyone else. Relative = stepping a bit to the left or right but still in line. Fixed = standing on a balcony overlooking the room — doesn't move even when people shift around. Absolute = standing next to a specific person (nearest positioned ancestor) — moves with that person. Sticky = starts in line but floats up to the ceiling when you walk past them.
Warning: position: fixed elements are removed from document flow and can overlap other content. Always test your layout after applying fixed positioning, and use z-index to control stacking order.

Summary

  • CSS (Cascading Style Sheets) separates presentation from structure. HTML provides content; CSS provides styling.
  • CSS Syntax: selector { property: value; } — declarations end with semicolons.
  • Selectors: Element (p), ID (#id), Class (.class), Descendant (A B), Grouped (A, B), Attribute ([attr]).
  • Three insertion methods: External (recommended), Internal, Inline (avoid).
  • Specificity: Inline > ID > Class/Attribute > Element. Equal specificity = last rule wins.
  • Box Model: Content → Padding → Border → Margin. Use box-sizing: border-box for intuitive sizing.
  • Colours: Named, HEX (#ff0000), RGBA (rgba(255,0,0,0.5)).
  • Text & Fonts: Use font shorthand, always include a generic family fallback.
  • Links: Style in order: :link, :visited, :hover, :active (LoVe HA).
  • Display: block (full width, new line), inline (in-line, no width/height), none (hidden, removed).
  • Position: static (default), relative (nudge from original), fixed (viewport, doesn't scroll), absolute (nearest positioned ancestor), sticky (relative then fixed on scroll).
