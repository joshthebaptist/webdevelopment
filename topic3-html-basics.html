Topic 3 of 14 — HTML Fundamentals (Part 1)

HTML Fundamentals (Part 1)

1. What is HTML?

Analogy: Think of HTML as the skeleton of a human body. Just as a skeleton gives the body its shape and structure, HTML gives a web page its structure. Without HTML, a web page would be formless — like a body without bones. CSS is like the skin and clothing (appearance), and JavaScript is like the muscles and brain (behavior). But HTML is the foundation everything else builds upon.

HTML stands for HyperText Markup Language. Let us break down every word:

Word Meaning
HyperText Text that contains links to other texts. When you click a link on a webpage and it takes you somewhere else, that is "hypertext" — text that goes beyond the current page.
Markup Data included in an electronic document. "Markup" means you annotate plain text with special codes (tags) to define how text should be structured or displayed. Think of it like a teacher marking up a student's essay with red pen — adding notes about structure and formatting.
Language A set of rules and grammar. Just like English has rules for sentences, HTML has rules for how you write tags and combine elements.
Note: A markup language is a set of rules governing what data may be included in a document and how it is combined with the content of the document. HTML is plain text, but includes a variety of tags that define the structure of the document. It defines the structure of text-based and multimedia information in a web page.
Key Fact: HTML is not a programming language like Python or Java. It is a markup language — it structures content but cannot perform logic (like calculations or decisions) on its own.

Why is HTML Easy to Learn?

HTML is relatively easy to learn because every tag is predefined. You do not invent your own tags — you use the ones that already exist in the HTML specification. Web browsers (Chrome, Firefox, Safari, Edge, etc.) read HTML and display the web page as output.

Tools You Need

You can write HTML in any simple text editor:

  • Notepad (Windows) or TextEdit (Mac)
  • Sublime Text (free, popular among developers)
  • Visual Studio Code (VSCode) (free, most popular today)

Save your files with a .html or .htm extension. Then open the file in any web browser to see your page.

2. A Brief History of HTML

Analogy: Think of HTML's history like the evolution of a car. The first car (HTML 1.0 in 1991) was a simple machine with just 18 parts. Over the decades, new models added features: better engines, air conditioning, GPS. HTML5 is like a modern electric car — sleek, powerful, and packed with features that the original could never do.
Year Milestone
1991 Tim Berners-Lee invented HTML while working at CERN (the European Organization for Nuclear Research). The first version had only 18 tags.
Late 1990s HTML grew rapidly through versions 2.0, 3.2, and 4.0 as the World Wide Web exploded in popularity.
2000s XHTML emerged, enforcing stricter rules (like requiring all tags to be closed). It was like a grammar teacher for HTML.
2014–2015 HTML5 was finalized. It replaced HTML 3.2, HTML 4, and XHTML with one unified standard. HTML5 introduced many new elements and APIs.
2016 HTML 5.2 improved security, added a Payment Request API, and enhanced accessibility features.
Today HTML has approximately 142 tags and continues to evolve.

3. Must-Have Elements (The Four Pillars)

Analogy: Every HTML page is like a letter you mail. The <!DOCTYPE html> is like the type of envelope (tells the post office what kind of mail this is). The <html> is the entire envelope itself. The <head> is the information written on the outside of the envelope (sender, subject line, metadata). The <body> is the actual letter inside — the content the reader sees.

Every single HTML page must have these four elements. Without them, the browser may display the page incorrectly.

Fill in the blanks: Complete the HTML skeleton below: 
<!DOCTYPE ______>
<______>
  <head>
    <title>My Page</title>
  </______>
  <body>
    <p>Hello World!</p>
  </body>
</______>

What each element does:

Element Purpose
<!DOCTYPE html> Tells the browser: "This document is HTML5." It must be the very first line. Without it, the browser switches to "quirks mode" and may render things incorrectly.
<html> The root element. Everything else goes inside this tag. It wraps the entire page.
<head> Contains metadata — information about the page that is NOT displayed directly on the page. This includes the title, character set, links to CSS, and more.
<body> Contains everything visible on the page — text, images, links, buttons, and all other content the user sees and interacts with.
Warning: Content placed outside the <body> tag will not be visible on the page. Always put your visible content inside <body>.

4. Hello World — Your First Web Page

Analogy: "Hello World" is the universal first program in every programming language — like the first chord a guitar student learns. It proves your setup works and gives you a starting point to build from.

Here is the simplest possible HTML page:

<!DOCTYPE html>
<html>
<head>
    <title>My First Page</title>
</head>
<body>
    Hello World!
</body>
</html>

Important Rules

1. HTML does NOT recognize extra white spaces.

If you write multiple spaces or press Enter multiple times in your HTML, the browser will collapse them all into a single space. HTML treats whitespace differently from a text editor.

Example: Writing Hello    World (with 4 spaces) displays as Hello World (1 space). To force extra spaces, use the &nbsp; entity (more on this later).

2. Comments

Comments are notes for yourself or other developers. The browser ignores comments completely — they do not appear on the page.

5. The Title Element

Analogy: The <title> is like the name tag you wear at a conference. It appears at the top (in the browser tab), helps people identify you (bookmarks), and helps organizers find you (search engines).

The <title> element lives inside the <head> and serves three critical purposes:

Purpose Description
Browser Tab The title text appears on the browser tab — the small label you see when multiple tabs are open.
Bookmarks/Favorites When a user bookmarks your page, the title is used as the default bookmark name.
Search Engines Search engines like Google use the title as the blue clickable link in search results. A good title improves your page's visibility. 
<head>
    <title>Topic 3: HTML Fundamentals - TP2543</title>
</head>

6. Paragraphs and Headings

Analogy: Headings are like the headings in a textbook: H1 is the book title, H2 is the chapter title, H3 is the section heading, and so on. Paragraphs are the body text of each section. Just like a well-organized textbook, your web page needs clear hierarchy.

Headings: <h1> through <h6>

HTML provides six levels of headings. <h1> is the most important (largest), and <h6> is the least important (smallest).

<h1>Main Title (largest)</h1>
<h2>Sub-Heading</h2>
<h3>Sub-sub-Heading</h3>
<h4>Heading Level 4</h4>
<h5>Heading Level 5</h5>
<h6>Heading Level 6 (smallest)</h6>

h1 — Main Title

h2 — Chapter Title

h3 — Section Heading

h4 — Sub-section

h5 — Minor Heading
h6 — Smallest Heading
Best Practice: Use only one <h1> per page (it is the main title). Do not skip heading levels (e.g., do not jump from <h2> to <h5>). Search engines and screen readers rely on proper heading hierarchy.

Paragraphs: <p>

The <p> tag defines a paragraph of text. Browsers add a small space above and below each paragraph automatically.

<p>This is the first paragraph of text. It will be displayed
as a block of text with some space above and below it.</p>

<p>This is the second paragraph. Each paragraph starts on a
new line with spacing between them.</p>

Block-Level vs Inline Elements

Analogy: A block-level element is like a paragraph in a book — it always starts on a new line and takes up the full width (like a road that spans the entire width of a page). An inline element is like a word highlighted in yellow — it stays inside the line and only takes up as much space as it needs (like a sticker placed on the road without blocking traffic).
Type Behavior Examples
Block-level Takes up the full width available. Always starts on a new line. Height and width can be set. <div>, <p>, <h1>–<h6>, <table>, <ul>, <li>
Inline Only takes up as much width as needed. Does NOT start on a new line. Width and height cannot be set. <span>, <a>, <img>, <b>, <i>, <em>

7. Hyperlinks

Analogy: Hyperlinks are like doors in a building. Each door (link) takes you to a different room (web page). The href attribute is the address written on the door. You can also have doors that open in a new building (_blank), or doors that lead to a specific room in the same building (#id).

The <a> (anchor) tag creates hyperlinks. The href attribute specifies the destination.

Basic Syntax

<a href="https://www.google.com">Visit Google</a>

Types of Links

1. Link to a remote page:

<a href="https://www.example.com">Visit Example</a>

2. Link to a file (e.g., a PDF):

<a href="documents/brochure.pdf">Download Brochure</a>

3. Link to an email address:

<a href="mailto:info@example.com">Send us an Email</a>

Clicking this link opens the user's default email client.

4. Internal link (within the same page):

<a href="#section2">Jump to Section 2</a>

<!-- Later on the same page: -->
<h2 id="section2">Section 2</h2>

Target Attribute

The target attribute controls where the linked page opens:

Value Behavior
_blank Opens the link in a new tab or window. Most commonly used.
_self Opens the link in the same tab (this is the default behavior).
_parent Opens the link in the parent frame (used with iframes/frames).
_top Opens the link in the full body of the window, canceling all frames. 
<a href="https://www.google.com" target="_blank">Open Google in new tab</a>

8. Images

Analogy: The <img> tag is like placing a picture frame on a wall. The src (source) is the actual photograph inside the frame. The alt text is the label card you place below the frame in a museum — it describes the picture for people who cannot see it.

Basic Syntax

<img src="photo.jpg" alt="A beautiful sunset" width="600" height="400">

Supported Image Formats

Format Extension Best For
JPEG/JPG .jpg, .jpeg Photographs, complex images with many colors. Lossy compression (smaller file size, some quality loss).
GIF .gif Simple animations, images with few colors. Supports transparency but limited to 256 colors.
PNG .png Images needing transparency, logos, graphics. Better quality than JPEG but larger file size.
SVG .svg Scalable vector graphics — logos, icons, illustrations. Can be resized infinitely without quality loss.

The alt Attribute (Accessibility)

The alt attribute provides alternative text for the image. It is displayed when:

  • The image fails to load (broken link)
  • A blind user uses a screen reader to browse the page
  • The user has images disabled in their browser
Important: Always include the alt attribute. It is required by the HTML specification and essential for accessibility.

Image as a Hyperlink

Wrap an <img> tag inside an <a> tag to make the image clickable:

<a href="https://www.example.com">
    <img src="logo.png" alt="Company Logo">
</a>

Image Maps

An image map lets you define clickable areas on a single image, each linking to a different destination:

<img src="worldmap.jpg" alt="World Map" usemap="#mymap" width="600" height="400">

<map name="mymap">
    <area shape="rect" coords="50,50,200,200" href="europe.html" alt="Europe">
    <area shape="circle" coords="400,300,80" href="asia.html" alt="Asia">
    <area shape="poly" coords="100,300,200,350,150,400" href="africa.html" alt="Africa">
</map>
Note: Image maps are considered a legacy technique. In modern web development, you would typically use separate smaller images or CSS/JavaScript for clickable regions.

9. Absolute vs Relative File Paths

Analogy: An absolute path is like giving someone your full postal address: "123 Main Street, Springfield, IL 62701, USA" — it works from anywhere in the world. A relative path is like saying "it is two doors down from here" — it only works if you are starting from the right place.
Type Format Example
Absolute Full URL including protocol (http:// or https://) https://www.example.com/images/photo.jpg
Relative Path relative to the current file's location images/photo.jpg or ../images/photo.jpg

Relative Path Examples

Folder structure analogy: Imagine your website is a filing cabinet: 
website/
├── index.html
├── about/
│   └── about.html
├── images/
│   └── logo.png
└── css/
    └── style.css
  • From index.html to logo.png: images/logo.png
  • From about.html to logo.png: ../images/logo.png (go up one folder, then into images)
  • ../ means "go up one directory level"
  • ./ means "current directory"
Best Practice: Always use relative paths for links within your website. This way, your site works on any server without needing to change URLs. Only use absolute paths for linking to external websites.

10. Special Characters (HTML Entities)

Analogy: HTML entities are like escape codes. Some characters have special meaning in HTML (like < and > which define tags). If you want to display these characters as actual text on the page, you need to "escape" them using entities — similar to how you would tell someone, "I am talking about the symbol <, not the action of opening a tag."

Some characters cannot be typed directly into HTML because they have special meanings. Use HTML entities instead:

Entity Code Display Description
&amp; &amp; & Ampersand
&lt; &lt; < Less than
&gt; &gt; > Greater than
&nbsp; &nbsp;   Non-breaking space (forces a space)
&copy; &copy; © Copyright symbol
&quot; &quot; " Quotation mark
&apos; &apos; ' Apostrophe

Decimal and Hexadecimal Codes

You can also use numeric codes to display any Unicode character:

<p>Copyright &#169; 2024 My Website</p>       <!-- Decimal -->
<p>Copyright &#xA9; 2024 My Website</p>       <!-- Hexadecimal -->

Both display: Copyright © 2024 My Website

11. Horizontal Rules

The <hr> tag creates a horizontal line (thematic break) to separate content sections:

<p>First section of content.</p>

<hr>

<p>Second section of content.</p>

This renders as:

First section of content.

Second section of content.

Note: The <hr> tag is a legacy element. In modern web development, you should prefer using CSS (e.g., border-top or ::after pseudo-elements) to create dividers, as CSS gives you full control over the line's appearance.

12. Lists

Analogy: Lists are like grocery lists. An unordered list is like a shopping list where order does not matter (you can buy milk or bread first). An ordered list is like a recipe — the steps must happen in order (step 1, step 2, step 3).

Unordered List (<ul>)

Items are marked with bullets. The order of items does not matter.

<ul>
    <li>HTML</li>
    <li>CSS</li>
    <li>JavaScript</li>
</ul>
  • HTML
  • CSS
  • JavaScript

Ordered List (<ol>)

Items are numbered automatically. Order matters.

<ol>
    <li>Step 1: Write the HTML</li>
    <li>Step 2: Add the CSS</li>
    <li>Step 3: Write the JavaScript</li>
</ol>
  1. Step 1: Write the HTML
  2. Step 2: Add the CSS
  3. Step 3: Write the JavaScript

You can place lists inside other lists to create sub-categories:

<ul>

13. Tables

Analogy: An HTML table is like a spreadsheet. Rows go left to right, columns go up and down. <th> cells are bold header labels (like column titles in Excel), and <td> cells are the regular data cells.

Basic Table Structure

<table>
    <caption>Student Grades</caption>
    <thead>
        <tr>
            <th>Name</th>
            <th>Grade</th>
        </tr>
    </thead>

            <td>Alice</td>
            <td>A</td>
        </tr>

            <td>Bob</td>
            <td>B+</td>
        </tr>
    </tbody>
    <tfoot>
        <tr>
            <td>Average</td>
            <td>A-</td>
        </tr>
    </tfoot>
</table>

Table Elements Explained

Element Purpose
<table> Creates the table container
<caption> Title/label for the table (appears above the table)
<thead> Groups the header rows
<tbody> Groups the body/content rows
<tfoot> Groups the footer rows (like totals or summaries)
<tr> A table row (goes left to right)

Merging Cells

colspan merges cells horizontally (across columns):

<tr>
    <td colspan="2">This cell spans 2 columns</td>

rowspan merges cells vertically (across rows):

<tr>
    <td rowspan="2">This cell spans 2 rows</td>
    <td>Cell 2</td>

    <td>Cell 3</td>
</tr>

Rowspan = 2 (spans down) Row 2 Row 2

14. Div vs Span

Analogy: A <div> is like a shipping box — it is a container that holds multiple items and takes up the entire shelf. A <span> is like a label sticker — you stick it on one item inside the box to highlight or identify it. Both are containers, but at different scales.
Element Type Purpose
<div> Block-level Groups large sections of content. Takes up full width. Used for layout and structure.
<span> Inline Groups a small piece of content within a line. Only takes up as much width as needed. Used for styling specific words or phrases. 
<div style="background: lightblue; padding: 10px;">
    <p>This is a <span style="color: red; font-weight: bold;">highlighted word</span> in a paragraph.</p>
</div>
Note: Both <div> and <span> have no visual effect by themselves. They are purely structural containers. You typically style them using CSS.

15. Inline Text Formatting Tags

Analogy: These tags are like tools in a writer's toolkit. <b> is a highlighter pen (makes text bold). <i> is an italicizer (slants text). <strong> is like underlining a word in red — it means "this is really important." <em> is like adding stress to your voice when speaking.
Tag Purpose Visual Result
<b> Bold text (visual emphasis only) This is bold
<i> Italic text (visual emphasis only) This is italic
<em> Emphasized text (semantic — tells browsers/screen readers this is important) This is emphasized
<strong> Strong importance (semantic — stronger than <b>) This is strong
<u> Underlined text This is underlined
<s> Strikethrough text (no longer relevant) This is struck
<mark> Highlighted text (like a yellow marker) This is highlighted
<sub> Subscript text (like H2O) H2O
<sup> Superscript text (like x2) x2
Important Distinction: <b> and <strong> both make text bold, but <strong> carries semantic meaning — it tells the browser and screen readers that the text has extra importance. Similarly, <i> vs <em>. Always prefer the semantic versions (<strong>, <em>) for better accessibility.

16. Meta Elements

Analogy: Meta tags are like the resume summary at the top of a job application. The hiring manager (search engine) does not see this summary on the main resume page, but it helps them quickly understand what the resume is about, who wrote it, and how to categorize it.

<meta> tags go inside <head> and provide metadata — data about the page that is not visible to users but is used by browsers, search engines, and other web tools.

Common Meta Tags

<head>
    <!-- Character encoding -->
    <meta charset="UTF-8">

    <!-- Page description for search engines -->
    <meta name="description" content="Learn HTML fundamentals from scratch">

    <!-- Keywords for search engines -->
    <meta name="keywords" content="HTML, web development, beginner">

    <!-- Author of the page -->
    <meta name="author" content="John Doe">

    <!-- Software that generated the page -->
    <meta name="generator" content="VSCode">

    <!-- Auto-refresh the page every 30 seconds -->
    <meta http-equiv="refresh" content="30">

    <!-- Responsive design viewport -->
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
</head>

Key Meta Tags Explained

Meta Tag Purpose
charset="UTF-8" Sets the character encoding. UTF-8 supports virtually all characters and symbols in the world. Always include this.
name="description" A short description of the page. Search engines display this below the title in search results.
name="keywords" Comma-separated keywords related to the page. Less important today (search engines rely less on this).
name="author" The author of the web page.
http-equiv="refresh" Automatically redirects or refreshes the page. content="30" means refresh every 30 seconds. You can also redirect: content="5;url=https://newsite.com" (redirect after 5 seconds).
name="viewport" Controls how the page is displayed on different devices. Essential for responsive design.

Summary

  • HTML is the standard markup language for creating web pages. It provides the structure of a page.
  • Every page needs <!DOCTYPE html>, <html>, <head>, and <body>.
  • Tags define elements. Most come in opening/closing pairs (<tag>...</tag>); some are empty (<br>, <hr>, <img>).
  • Block-level elements (div, p, h1-h6) take full width. Inline elements (span, a, b, i) take only needed width.
  • Headings (h1-h6) create hierarchy. Paragraphs (p) hold body text.
  • Hyperlinks (<a href="...">) connect pages. Images (<img src="..." alt="...">) embed visuals.
  • Lists organize content: <ul> for bullets, <ol> for numbers, <li> for items.
  • Tables display tabular data with <table>, <tr>, <th>, <td>.
  • <div> is a block-level container; <span> is an inline container.
  • HTML entities display special characters (&amp; &lt; &gt; &nbsp;).
  • Meta tags provide invisible metadata (charset, description, keywords) for browsers and search engines.
  • Use relative paths for internal links. Use semantic tags (<strong>, <em>) over visual ones (<b>, <i>).
