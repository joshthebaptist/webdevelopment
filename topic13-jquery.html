Topic 13: jQuery — Write Less, Do More

1. What is jQuery?

Analogy — The Universal Remote: Imagine you have a dozen different devices in your living room — a TV, speakers, a DVD player, a projector. Instead of learning every individual remote control, you buy a universal remote that speaks every device's language. jQuery is that universal remote for your web page. Instead of writing verbose, browser-specific JavaScript to manipulate the DOM, handle events, or animate elements, jQuery gives you one clean, simple interface to do it all.

jQuery is a fast, small, and feature-rich JavaScript library. It makes the following tasks dramatically simpler:

  • HTML document traversal — finding elements on the page
  • DOM manipulation — adding, removing, or changing elements
  • Event handling — responding to clicks, hovers, key presses, etc.
  • Animation — showing, hiding, fading, and sliding elements
  • Ajax — communicating with servers without page reloads

Created by John Resig in 2006, its motto is "Write less, do more."

Key Fact: Over 70% of the 10 million most popular websites use jQuery. Major companies like Google, Microsoft, IBM, and Netflix rely on it.

2. jQuery Syntax

Analogy — The Command Sentence: Think of jQuery syntax like giving instructions to a very capable assistant. The formula is: $(who).what(). You say who you want to act on (the selector), and what you want done (the action). "Hey jQuery, hide the image." → $("img").hide().

The Core Pattern

$(selector).action()
  • $ — The jQuery function (alias for jQuery)
  • selector — Finds HTML elements on the page
  • action() — The operation to perform on those elements

Examples

CodeMeaning
$(this).hide()Hide the current element
$("img").hide()Hide all <img> elements
$(".test").hide()Hide all elements with class="test"
$("#test").hide()Hide the element with id="test"

3. The Document Ready Event

Analogy — Opening a Shop: You wouldn't start serving customers before the shop is fully open, lights are on, and shelves are stocked. Similarly, you should never manipulate the DOM before the HTML document is fully loaded and ready. The $(document).ready() event is jQuery's way of saying: "Wait until everything is loaded, then start working."

jQuery lets you detect when the document is fully loaded and safe to manipulate. There are two common syntaxes:

Full Version

$(document).ready(function() {
    // Your code here — runs when the DOM is ready
    console.log("Document is ready!");
});

Shorthand Version (Recommended)

$(function() {
    // Same thing, shorter syntax
    console.log("Document is ready!");
});
Tip: Both versions do the exact same thing. The shorthand $(function(){...}) is more concise and preferred in modern code.

6. Event Handling with .on()

Analogy — Subscribing to Notifications: The .on() method is like signing up for push notifications. You say "Notify me when this button is clicked," and jQuery registers your callback. When the event fires, your function runs.

Basic Event Binding

$("#myButton").click(function() {
    alert("Button was clicked!");
});

Using .on() — More Flexible

$("#myButton").on("click", function() {
    alert("Button was clicked!");
});

Multiple Events on One Element

$("#myElement").on("click mouseenter", function() {
    $(this).css("background-color", "yellow");
});

Event Delegation (for dynamic elements)

// Attaches handler to all current AND future .item elements
$("#parentList").on("click", ".item", function() {
    $(this).toggleClass("active");
});
Why .on() is preferred: It supports event delegation (handling events for elements that don't exist yet), can bind multiple handlers, and is the most consistent API across jQuery versions.

7. CSS Manipulation & Dimensions

The .css() Method

Get or set CSS properties on elements:

// Get a CSS value
var color = $("p").css("color");

// Set a single CSS property
$("p").css("color", "red");

// Set multiple CSS properties
$("p").css({
    "color": "blue",
    "font-size": "20px",
    "background-color": "#f0f0f0"
});

Dimension Methods

Analogy — Measuring a Box: Imagine a gift box inside a shipping box. The content is the gift. height() measures just the gift. innerHeight() includes wrapping paper (padding). outerHeight() includes the gift box itself (border). outerHeight(true) includes the shipping box too (margin).

8. jQuery Effects

Analogy — Stage Curtains: Effects are like theatrical stage techniques. hide() is dropping the curtain. show() is raising it. fadeIn() is a slow spotlight fade-in. slideDown() is a curtain lowering from above. jQuery gives you a full toolkit of visual transitions.

Show / Hide / Toggle

$("p").hide();          // Instantly hide
$("p").hide(500);       // Hide over 500 milliseconds

$("p").show();          // Show the element
$("p").show(1000);      // Show over 1 second

$("p").toggle();        // Toggle visibility
$("p").toggle(500);     // Toggle with animation

Fading Effects

$("p").fadeOut();            // Fade out (display: none)
$("p").fadeOut(1000);

$("p").fadeIn();             // Fade in
$("p").fadeIn(1000);

$("p").fadeToggle();         // Toggle with fade

$("p").fadeTo(1000, 0.5);   // Fade to a specific opacity

Sliding Effects

$("p").slideDown();          // Slide down (reveal)
$("p").slideDown(1000);

$("p").slideUp();            // Slide up (hide)
$("p").slideUp(1000);

$("p").slideToggle();        // Toggle with sliding
Note: hide() sets display: none. fadeOut() animates opacity to 0, then sets display: none. The visual result differs because fadeOut gives a smoother transition.

9. Custom Animation with .animate()

The animate() method lets you animate numeric CSS properties over time:

$("div").animate({
    width: "70%",
    height: "200px",
    marginLeft: "150px",
    opacity: 0.5
}, 1000);  // duration in milliseconds

Animatable Properties

You can animate any numeric CSS property:

  • width, height
  • margin, padding (all sides)
  • opacity
  • top, left, right, bottom
  • font-size, line-height
Limitation: Basic jQuery animate() cannot animate non-numeric CSS properties like color, background-color, or border-color. You need a plugin like jQuery Color for color animations.

Animation Queue Control

MethodPurpose
.stop()Stop the current animation
.finish()Stop all queued animations and jump to final state
.queue()Show or modify the animation queue
.clearQueue()Remove all queued animations
.delay(ms)Pause before next queued animation
.dequeue()Execute the next queued function

Chaining Animations

$("div")
    .animate({ height: "300px" }, 1000)
    .animate({ width: "300px" }, 1000)
    .fadeOut(500);

10. Method Chaining

Analogy — A Sentence of Actions: Chaining is like saying in one breath: "Walk to the store, pick up milk, pay for it, and walk back." Instead of writing four separate sentences (four lines of code), you connect them into one fluent sentence. Each action passes the result to the next.

jQuery allows you to chain multiple methods on the same element in a single statement. Each method returns the jQuery object, allowing the next method to be called on the same selection.

Without Chaining

$("h1").css("color", "red");
$("h1").addClass("title");
$("h1").slideDown(500);

With Chaining

$("h1")
    .css("color", "red")
    .addClass("title")
    .slideDown(500);

Real-World Example

$("#alertBox")
    .hide()
    .html("Form submitted successfully!")
    .addClass("alert-success")
    .fadeIn(500)
    .delay(2000)
    .fadeOut(500);
Why chain? Cleaner, more readable code. Fewer lines. Less repetition of selectors.

12. Bootstrap Modal Deep Dive

Analogy — The Popup Window: A modal is like a cashier's window that pops up in front of you. You must interact with it (or dismiss it) before you can return to the rest of the store. It "captures" your attention by dimming the background.

Modal HTML Structure

<!-- Button trigger -->
<button data-bs-toggle="modal" data-bs-target="#myModal">
  Open Modal
</button>

<!-- Modal -->
<div class="modal" id="myModal">
  <div class="modal-dialog">
    <div class="modal-content">

      <div class="modal-header">
        <h5 class="modal-title">Modal Title</h5>
        <button data-bs-dismiss="modal">&times;</button>
      </div>

      <div class="modal-body">
        <p>Modal content goes here.</p>
      </div>

      <div class="modal-footer">
        <button data-bs-dismiss="modal">Close</button>
        <button>Save Changes</button>
      </div>

    </div>
  </div>
</div>

Key Attributes

AttributePurpose
data-bs-toggle="modal"Tells Bootstrap this button opens a modal
data-bs-target="#myModal"Points to the modal element's ID
data-bs-dismiss="modal"On any element, clicking it closes the modal

Opening / Closing with jQuery

// Open modal programmatically
$("#myModal").modal("show");

// Close modal programmatically
$("#myModal").modal("hide");
Two ways to trigger:
1. Data attributes (no JavaScript needed): data-bs-toggle and data-bs-target
2. jQuery: $("#myModal").modal("show")

13. jQuery Plugins

Analogy — App Store for jQuery: Plugins are like apps you install on your phone. jQuery is the operating system, and plugins add new capabilities — a camera app, a calculator, a map. You install a plugin by including its JavaScript file, and suddenly jQuery gains new methods.

jQuery plugins are standard JavaScript files that extend jQuery with new methods. They follow a convention of adding to the $.fn prototype.

How to Include a Plugin

<head>
  <!-- jQuery first -->
  <script src="jquery-3.6.0.min.js"></script>
  <!-- Then the plugin -->
  <script src="jquery.plugin.js"></script>
</head>
Order matters: Always include the jQuery library before any plugins that depend on it.

DataTable Plugin Example

The DataTable plugin transforms a plain HTML table into a powerful, interactive listing with sorting, searching, and pagination.

<script src="https://cdn.datatables.net/1.13.4/js/jquery.dataTables.min.js"></script>

<script>
$(document).ready(function() {
    $("#myTable").DataTable();
});
</script>

14. Putting It All Together

Complete Example: Interactive FAQ Page

<!DOCTYPE html>
<html>
<head>
  <script src="jquery-3.6.0.min.js"></script>
  <style>
    .faq-answer { display: none; }
  </style>
</head>
<body>
  <h1>FAQ</h1>

  <div class="faq-item">
    <h3 class="faq-question">What is jQuery?</h3>
    <p class="faq-answer">A fast JavaScript library.</p>
  </div>

  <div class="faq-item">
    <h3 class="faq-question">Is it free?</h3>
    <p class="faq-answer">Yes, it is open source!</p>
  </div>

  <script>
    $(function() {
      // Click a question to toggle its answer
      $(".faq-question").on("click", function() {
        $(this)
          .next(".faq-answer")
          .slideToggle(300)
          .siblings(".faq-answer")
          .slideUp(300);
      });
    });
  </script>
</body>
</html>
What this demonstrates: Document ready, selectors, event handling (.on()), chaining, and effects (slideToggle, slideUp) — all core jQuery concepts working together.

What does the $ symbol represent in jQuery?

Why use $(document).ready()?
To ensure the DOM is fully loaded before running jQuery code, preventing errors from selecting non-existent elements.

Color properties (e.g., color, background-color). A plugin like jQuery Color is needed.

Modals, Dropdowns, Tooltips, Popovers, Carousels, Tabs, Collapse/Accordion, Alerts, and more.

Summary

  • jQuery is a lightweight JavaScript library that simplifies DOM manipulation, events, animations, and Ajax.
  • Syntax: $(selector).action() — the $ is the jQuery function.
  • Always wrap code in $(document).ready() or $(function(){...}) to ensure the DOM is loaded.
  • Selectors use CSS syntax plus jQuery extensions to find elements.
  • Event handling with .on() provides the most flexibility, including delegation.
  • Effects: hide, show, toggle, fade, slide — all with animation support.
  • .animate() handles custom animations for numeric CSS properties (not colors without plugins).
  • Chaining allows multiple methods in a single statement for cleaner code.
  • Bootstrap components rely on jQuery plugins for interactivity.
  • Plugins extend jQuery's capabilities; include them after the jQuery library.
