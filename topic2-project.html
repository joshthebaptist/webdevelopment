Topic 2: Course Project Overview

In this topic, you will understand the course project structure. You will learn how you are going to transform a 2-tier system from your previous course (TU2983) into a web-based 3-tier system through three guided project tasks.

1. From TU2983 to TP2543

In your previous course TU2983 (Advanced Database), you developed a 2-tier “ordering” system with 3 basic functionalities:

  1. Manage Products — Add, view, edit, and delete products.
  2. Make an Order — Select products and create an order.
  3. View an Order — Look up existing orders and their details.

In TP2543, you will take this same system — using the same theme, problems, data, and functionalities — and transform it into a web-based 3-tier system. This means adding a web browser interface (frontend) and a web server layer on top of your existing database and logic.

🏪 Analogy — From Food Truck to Full Restaurant:
  • 2-tier (Food Truck): You have a food truck. The cook (application logic) directly accesses the fridge (database) and hands food to the customer through a small window. It works, but it’s limited — only one window, limited menu, no dining area.
  • 3-tier (Full Restaurant): Now you upgrade to a full restaurant. You add a beautiful dining room (frontend/browser interface) where customers sit comfortably, browse a nice menu (HTML pages), and place orders with waiters (web server). The waiters take orders to the kitchen (application server), where chefs cook using ingredients from the pantry (database). Each tier has its own role, and the system is more scalable, maintainable, and professional.

🚛 2-Tier System

Client ↔ Database

The application logic runs directly on the client, communicating directly with the database. No web server in between.

Limited to desktop apps — not web-accessible.

🏠 3-Tier System

Presentation Tier ↔ Logic Tier ↔ Data Tier

Three separate layers: Frontend (browser), Backend (web + application server), and Database. Each tier communicates through defined interfaces.

Web-accessible, scalable, maintainable.

4. Understanding the 3-Tier Architecture

Your project follows the 3-tier architecture pattern, which separates your application into three logical layers:

💻 Presentation Tier (Frontend)
HTML, CSS — Runs in the browser
↓ HTTP ↓
⚙️ Logic Tier (Backend)
PHP scripts — Runs on the server (Apache)
↓ SQL ↓
🗄️ Data Tier (Database)
MySQL — Stores all application data
  • Presentation Tier — The HTML pages styled with CSS that users interact with in their browser. This tier handles how data is displayed and how users input information.
  • Logic Tier — The PHP scripts running on the Apache server. This tier processes user input, executes business logic (e.g., calculating order totals, validating data), and communicates with the database.
  • Data Tier — The MySQL database that stores all persistent data (products, orders, customers). This tier is responsible for data storage, retrieval, and integrity.
🏪 Analogy — The Restaurant Upgrade:
  • Presentation Tier = The dining room and menu — Customers see the menu, choose seats, and place orders with waiters.
  • Logic Tier = The kitchen — Chefs receive orders, follow recipes, coordinate preparation, and plate the food.
  • Data Tier = The pantry and storage — Where all ingredients are organized, stored, and retrieved as needed.

5. Key Terms — Flashcards

Click on a flashcard to flip it and reveal the definition.

2-Tier System
An application architecture with two layers: a client (presentation) and a database (data). The application logic runs directly on the client, communicating directly with the database. Typically a desktop application.
3-Tier System
An application architecture with three layers: Presentation Tier (frontend), Logic Tier (backend/application server), and Data Tier (database). Each tier has a distinct responsibility and communicates through defined interfaces.
CRUD
Create, Read, Update, Delete — The four basic operations performed on data. In this project: Manage Products (Create, Read, Update, Delete), Make an Order (Create), View an Order (Read).
Presentation Tier
The frontend layer of a 3-tier system. It handles the user interface and user experience, built with HTML, CSS, and possibly JavaScript. Runs in the user’s browser.
Logic Tier
The backend layer of a 3-tier system. It processes business logic, handles data validation, performs calculations, and mediates between the presentation and data tiers. In this course, it runs PHP on Apache.
Data Tier
The database layer of a 3-tier system. It manages persistent data storage and retrieval. In this course, it is a MySQL database accessed via SQL queries from PHP.
XAMPP
A free, open-source web server solution stack that includes Apache (web server), MySQL (database), PHP, and Perl. Used for local web development and testing.
Ordering System
The core project application with 3 functionalities: (1) Manage Products, (2) Make an Order, (3) View an Order. This is transformed from a 2-tier desktop app to a 3-tier web application.
Self-Directed Task
Each project task is self-directed, meaning you are guided by written instructions but must work independently to implement the requirements, debug issues, and produce the final deliverable.
